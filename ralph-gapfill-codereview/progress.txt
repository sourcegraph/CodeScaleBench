## Codebase Patterns
- Code review tasks: benchmarks/ccb_codereview/{task_id}/ — existing suite, expand from 3 to 8 tasks
- Existing tasks: cr-aspnetcore-001, cr-calcom-001, cr-ghost-001 — study their structure as template
- Key files per task: task.toml, instruction.md, environment/Dockerfile, environment/inject_defects.sh, tests/test.sh, tests/expected_defects.json, tests/expected_patches/
- inject_defects.sh: uses sed to introduce bugs into clean code at specific lines
- expected_defects.json: [{file, line, severity, description, fix_description}]
- expected_patches/: one .patch file per defect showing the correct fix
- test.sh hybrid scoring: 50% detection F1 (match reported defects to expected by file path) + 50% fix quality (check if agent's code changes match expected patches via pattern matching)
- Agent output: code fixes applied directly + /workspace/review.json with [{file, line, severity, description, fix_applied}]
- Security defects: weight 2x in scoring (modify test.sh to double-weight critical/high severity)
- instruction.md must have ZERO MCP/Sourcegraph mentions
- Config: add to configs/codereview_2config.sh TASK_SG_REPO_NAMES array (not TASKS)
- Config: also add to configs/selected_benchmark_tasks.json with task_id, benchmark, language, difficulty, mcp_benefit_score
- Config: update total_selected count in selected_benchmark_tasks.json metadata
- inject_defects.sh uses python3 string replacement (not sed) for multi-line code changes
- Defect injection approach: read file → find exact string → replace → write back (python3 -c)
- For multi-defect files, inject_defects.sh runs separate python3 blocks per defect to avoid ordering issues
- Terraform v1.10.3 is a good pin for Go evaluation pipeline tasks — stable evaluate/eval_variable/context_apply/hook APIs
- Go repos use tabs for indentation — python3 triple-quoted strings in inject_defects.sh must use \t
- hashicorp/terraform is NOT indexed in Sourcegraph — use raw GitHub URLs for code verification
- For Go repos, NilHook pattern is excellent for cross-file defects — all hook embedders affected by base change
- Envoy v1.33.0 is a good pin for C++ filter chain tasks — has stable fault/ext_authz/filter_manager APIs
- VS Code 1.96.0 is a good pin for TypeScript editor core tasks — stable range/position/search/registry APIs
- VS Code uses tabs (not spaces) for indentation — inject_defects.sh must use \t in python triple-quoted strings
- VS Code has a typo "leftIsWordBounday" (not "Boundary") in textModelSearch.ts — must match exactly
- For TypeScript repos, fix verification regex must handle both `<` and `<=` patterns carefully (use negative lookahead or two-step check)
- curl-8_12_0 is a good pin for C security review tasks — stable openssl.c/cookie.c/transfer.c/base64.c APIs
- curl/curl IS indexed in Sourcegraph — use Sourcegraph for exact code verification at pinned tags
- For security-weighted scoring, use dict {defect_id: (passed, weight)} → weighted_hits/total_weight (not simple count)
- Preprocessor-guarded checks (#if SIZEOF_SIZE_T == 4) are excellent security defects — easy to remove as "cleanup" but critical on specific platforms
