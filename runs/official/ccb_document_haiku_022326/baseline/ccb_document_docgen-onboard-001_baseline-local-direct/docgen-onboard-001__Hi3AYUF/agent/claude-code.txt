This is a bidirectional gRPC streaming protocol.\n\n#### High-Level Data Flow\n\n```\n1. Kubernetes API (Services, Pods, Endpoints, CRDs)\n                    ↓\n2. Service Discovery Layer (pilot/pkg/serviceregistry/)\n   - Watches Kubernetes resources\n   - Builds internal Service and ServiceInstance models\n                    ↓\n3. Configuration Ingestion (pilot/pkg/config/)\n   - ConfigStore aggregates resources from multiple sources\n   - PushContext creates immutable snapshots of config state\n                    ↓\n4. Configuration Translation (pilot/pkg/networking/)\n   - Generators convert high-level config to Envoy protobuf format\n   - RouteGenerator, ClusterGenerator, EndpointGenerator, etc.\n   - Caching layer for performance (stores encoded protobufs)\n                    ↓\n5. xDS Server (pilot/pkg/xds/)\n   - Receives ADS (Aggregated Discovery Service) connections from Envoy\n   - Pushes updated configuration resources: Listeners, Routes, Clusters, Endpoints\n   - Handles incremental updates (only changed resources)\n                    ↓\n6. Envoy Proxy\n   - Receives configuration via xDS\n   - Applies configuration to dataplane\n   - Sends ACK/NACK to acknowledge receipt\n```\n\n#### Key xDS Resource Types\n\n- **CDS (Cluster Discovery Service)**: Defines upstream services (clusters) that Envoy can route to\n- **EDS (Endpoint Discovery Service)**: Lists actual service endpoints (IP:port pairs) for load balancing\n- **RDS (Route Discovery Service)**: HTTP routing rules for traffic management\n- **LDS (Listener Discovery Service)**: Listeners that Envoy binds to for incoming traffic\n- **SDS (Secret Discovery Service)**: TLS
