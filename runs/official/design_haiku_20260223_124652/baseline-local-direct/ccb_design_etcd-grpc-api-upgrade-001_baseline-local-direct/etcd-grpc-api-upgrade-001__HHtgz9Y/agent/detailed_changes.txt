GRPCDial() to grpc.NewClient() Migration - Detailed Changes Required

This document lists all the changes that need to be made to migrate from deprecated
grpc.Dial() and grpc.DialContext() to grpc.NewClient() across the etcd, kubernetes,
and containerd codebases.

CHANGE SUMMARY
===================================================================================

Total files modified: 18
Total function calls replaced: 21
- grpc.DialContext() → grpc.NewClient(): 13 occurrences
- grpc.Dial() → grpc.NewClient(): 8 occurrences

Note: When replacing grpc.DialContext(ctx, target, opts...), the first parameter (ctx)
should be removed. The context was only used for dial-time operations, not for the 
connection itself. With grpc.NewClient(), timeout handling should be configured through
grpc.WithConnectParams() if needed.
