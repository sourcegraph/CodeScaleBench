WHY THIS REFACTORING:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Current Problem:
  The class is named "RecordAccumulator" but it manages BATCHES of records,
  not individual records. This is misleading to developers reading the code.

Evidence:
  • Core data structure: ConcurrentMap<TopicPartition, Deque<ProducerBatch>>
  • Key methods operate at batch level: ready(), drain(), append()
  • Return types describe batches: RecordAppendResult, ReadyCheckResult

Solution:
  Rename to "BatchAccumulator" which better describes its responsibility.

Benefit:
  Improved code clarity and maintainability. Developers will immediately
  understand that this class manages batches for efficient network transmission.
