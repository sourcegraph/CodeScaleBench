## Key Implementation Details

### FixComponent (@Component("fix"))
- Extends DefaultComponent
- Manages component-level configuration
- Creates FixEndpoint instances
- Supports URI scheme: fix:sessionID?options
- Lazy-initializes configuration via doInit()

### FixEndpoint (@UriEndpoint scheme="fix")
- Extends DefaultEndpoint
- Creates Consumer and Producer instances
- Supports configuration binding via @UriParam
- Maintains endpoint-specific configuration

### FixConsumer
- Extends DefaultConsumer
- Implements lifecycle hooks (doStart/doStop)
- Routes FIX messages into Camel exchanges
- Sets FIX-specific headers on messages

### FixProducer
- Extends DefaultAsyncProducer
- Implements async message processing
- Extracts message content and metadata from exchanges
- Handles errors and exceptions appropriately

### FixConfiguration (@UriParams)
- POJO with @UriParam annotations for all configuration options
- Cloneable for endpoint-specific configuration copies
- Includes: sessionID, configFile, senderCompID, targetCompID, fixVersion, heartBeatInterval, socketConnectHost, socketConnectPort

### FixConstants
- Defines header constants: FIX_MESSAGE_TYPE, FIX_SESSION_ID, FIX_SENDER_COMP_ID, FIX_TARGET_COMP_ID
- Uses @Metadata annotations for documentation
