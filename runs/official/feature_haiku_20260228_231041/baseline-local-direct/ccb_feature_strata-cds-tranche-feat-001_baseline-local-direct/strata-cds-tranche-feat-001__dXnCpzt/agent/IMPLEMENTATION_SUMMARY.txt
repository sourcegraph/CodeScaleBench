================================================================================
PRICING MODEL
================================================================================

The IsdaCdsTranchePricer extends ISDA methodology with tranche adjustments:

1. LOSS ALLOCATION
   Formula: loss_tranche = max(0, min(loss_total, detachment) - attachment)
   
   Example (equity tranche with attachment=0%, detachment=3%):
   - If index loss = 2.5%, tranche loss = 2.5%
   - If index loss = 5.0%, tranche loss = 3.0%
   - If index loss = 1.0%, tranche loss = 1.0%

2. PROTECTION LEG
   - Computed using index survival probabilities
   - Weighted by attachment/detachment subordination
   - Accounts for recovery rate and daycount conventions

3. PREMIUM LEG
   - Risky PV01 (RPV01) weighted by tranche notional
   - Premium = RPV01 × fixedRate
   - Adjusted for paymentOnDefault treatment

4. PRICING EQUATION
   Price = (ProtectionLeg - RPV01 × FixedRate) / NotionalFactor
   PV = NotionalAmount × BuySell × Price

================================================================================
USAGE EXAMPLE
================================================================================

// Create equity tranche (0-3%)
CdsTranche equityTranche = CdsTranche.builder()
    .buySell(BuySell.BUY)
    .underlyingIndex(cdsIndex)
    .attachmentPoint(0.00)
    .detachmentPoint(0.03)
    .currency(Currency.USD)
    .notional(1_000_000)
    .fixedRate(0.075)
    .build();

// Create trade
CdsTrancheTrade trade = CdsTrancheTrade.builder()
    .info(TradeInfo.of(LocalDate.of(2024, 1, 15)))
    .product(equityTranche)
    .build();

// Price it
ResolvedCdsTrancheTrade resolved = trade.resolve(refData);
IsdaCdsTranchePricer pricer = IsdaCdsTranchePricer.DEFAULT;
CurrencyAmount pv = pricer.presentValue(resolved.getProduct(), rates, refDate, 
                                        PriceType.CLEAN, refData);

VALIDATION & CONSTRAINTS
================================================================================

CdsTranche Validation:
- attachmentPoint ∈ [0.0, 1.0]
- detachmentPoint ∈ [0.0, 1.0]
- attachmentPoint ≤ detachmentPoint
- notional > 0
- fixedRate ≥ 0
- underlyingIndex not null
- currency not null

ResolvedCdsTranche Validation:
- paymentPeriods not empty
- protectionEndDate after accrual end date
- All required fields present and valid

Trade Validation:
- info not null
- product (CdsTranche) not null
- upfrontFee amount currency matches product currency
