PROJECT STRUCTURE
================================================================================

The following three files need to be created in the flink-streaming-java module:

1. TradingSessionExtractor.java
   Location: flink-streaming-java/src/main/java/org/apache/flink/streaming/api/windowing/assigners/
   Purpose:  Functional interface for extracting market IDs from stream elements

2. PricingSessionWindow.java
   Location: flink-streaming-java/src/main/java/org/apache/flink/streaming/api/windowing/assigners/
   Purpose:  Window assigner that groups trading events by market session boundaries

3. PricingSessionTrigger.java
   Location: flink-streaming-java/src/main/java/org/apache/flink/streaming/api/windowing/triggers/
   Purpose:  Trigger that fires at market close (window end) via event-time timer

IMPLEMENTATION VERIFICATION
================================================================================

✓ PricingSessionWindow
  - Extends MergingWindowAssigner<Object, TimeWindow>
  - Implements assignWindows() to return TimeWindow for trading session
  - Implements mergeWindows() delegating to TimeWindow.mergeWindows()
  - Supports market ID, timezone, and session hours parameters
  - Factory method: forMarket(String, ZoneId, LocalTime, LocalTime)
  - Returns EventTimeTrigger as default trigger
  - Handles normal sessions, pre-market, and post-market events
  - Supports overnight sessions (futures markets)

✓ PricingSessionTrigger
  - Extends Trigger<Object, TimeWindow>
  - Fires at market close (window.maxTimestamp())
  - Registers event-time timers via TriggerContext
  - Implements canMerge() returning true
  - Implements onMerge() with timer re-registration
  - Implements clear() for proper resource cleanup
  - Handles late arrivals by immediate firing if watermark past window

✓ TradingSessionExtractor
  - Functional interface with String extract(T element) method
  - Extends Serializable for distribution
  - Modeled after SessionWindowTimeGapExtractor pattern
  - Enables dynamic market ID extraction from stream elements

USAGE EXAMPLES
================================================================================

1. Basic usage with PricingSessionWindow:

   PricingSessionWindow assigner = PricingSessionWindow.forMarket(
       "NYSE",
       ZoneId.of("America/New_York"),
       LocalTime.of(9, 30),
       LocalTime.of(16, 0)
   );

   KeyedStream<Trade, String> trades = ...;
   WindowedStream<Trade, String, TimeWindow> windowed =
       trades.window(assigner);

2. Using with custom trigger:

   windowed.trigger(PricingSessionTrigger.create())
           .apply(new AggregationFunction());

3. Multiple markets with TradingSessionExtractor (future enhancement):

   // Can be extended to support dynamic market ID extraction
   TradingSessionExtractor<Trade> extractor = trade -> trade.getMarketId();

ARCHITECTURE NOTES
================================================================================

Design follows established Flink patterns:
- Mirrors EventTimeSessionWindows for window assignment pattern
- Mirrors EventTimeTrigger for trigger implementation pattern
- Reuses TimeWindow for compatibility and merging
- Delegates to TimeWindow.mergeWindows() for standard interval merging
- Uses ZonedDateTime for timezone-aware time calculations

Key architectural decisions:
1. Timezone handling via java.time.ZoneId for DST support
2. Event-time based (isEventTime() returns true)
3. Supports both normal and overnight sessions
4. Proper watermark comparison for early firing
5. Full support for window merging (required for session windows)
