================================================================================
                         DESIGN PRINCIPLES
================================================================================

1. Composition Over Inheritance
   - CdsTranche contains CdsIndex rather than extending it
   - Allows independent evolution of tranche logic
   - Cleaner separation of concerns

2. Loss-on-Loss Pricing
   - Industry-standard CDO tranche methodology
   - Mathematically sound and well-tested
   - Delegates to proven ISDA CDS pricer for base calculations

3. Immutability
   - All products and trades are immutable
   - Thread-safe throughout
   - Builder pattern for construction

4. Delegation
   - Pricer delegates to existing IsdaCdsProductPricer
   - Calculation function delegates to measure class
   - No code duplication

5. Pattern Consistency
   - Follows existing Cds/CdsIndex/CdsTrade patterns
   - Uses Joda-Beans for all product/trade classes
   - Integrates with standard Strata calc engine
