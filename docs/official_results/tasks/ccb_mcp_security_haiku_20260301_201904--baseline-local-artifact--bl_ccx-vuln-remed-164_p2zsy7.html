## Key Findings

### 1. **Per-Route Bypass Configuration**
The HTTP filter implements per-route bypass through:
- `FilterConfigPerRoute` class with a `disabled()` method
- `getPerRouteFlags()` function that retrieves the skip_check flag
- `decodeHeaders()` function that checks if a route has disabled ext_authz and returns immediately

### 2. **Authorization Decision Caching**
The filter implements route cache control (not response caching):
- `clearRouteCache` flag allows cache invalidation when headers/query parameters are modified
- Route cache is cleared in `onComplete()` when authorization succeeds and modifications are made

### 3. **Immediate Response Path**
Multiple sendLocalReply paths for immediate denial:
- **Disabled filter**: Returns error when globally disabled with `deny_at_disable`
- **Authorization denied**: Sends response with authz server's status code and body
- **Authorization error**: Sends error response when authz service fails
- **Invalid response**: Rejects with HTTP 500 when response contains invalid headers
