## Overview

This release introduces significant API improvements focused on improving the evaluation service architecture, expanding authentication options, and enhancing segment handling with multi-segment support. The changes reflect Flipt's evolution toward a more flexible and scalable feature flag management system.

## Breaking Changes

> These changes require action from users upgrading. Refer to the Upgrade Guide for detailed migration instructions.

### Evaluation API Architecture Redesign

**Affected Components:** gRPC Evaluation Service, REST API

The legacy `Evaluate` and `BatchEvaluate` RPC methods in the core `Flipt` service are now **deprecated** in favor of the new dedicated `EvaluationService`.

- **Old API** (deprecated): `rpc Evaluate(EvaluationRequest) returns (EvaluationResponse)` and `rpc BatchEvaluate(BatchEvaluationRequest) returns (BatchEvaluationResponse)` in the `flipt.Flipt` service
- **New API** (recommended): Use the `flipt.evaluation.EvaluationService` with separate methods:
  - `rpc Boolean(EvaluationRequest) returns (BooleanEvaluationResponse)`
  - `rpc Variant(EvaluationRequest) returns (VariantEvaluationResponse)`
  - `rpc Batch(BatchEvaluationRequest) returns (BatchEvaluationResponse)`

**REST Endpoints:**
- `POST /evaluate/v1/boolean` — Evaluate a boolean flag
- `POST /evaluate/v1/variant` — Evaluate a variant flag
- `POST /evaluate/v1/batch` — Batch evaluate multiple requests

**Impact:** The new API provides stronger type safety (separate response types for boolean vs. variant evaluations) and a cleaner separation of concerns. Clients must migrate to the new endpoints.
