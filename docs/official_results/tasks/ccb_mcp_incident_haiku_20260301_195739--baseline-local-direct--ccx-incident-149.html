## Firefox CSS calc() Overflow Vulnerability - Key Findings

I identified the C++ and Rust source files involved in parsing CSS calc() expressions and handling numeric overflow:

### **Parsing Files:**
1. **`servo/components/style/values/specified/calc.rs`** - Parses CSS calc() syntax and supports various math functions (calc, min, max, clamp, round, mod, rem)
2. **`servo/components/style/values/generics/calc.rs`** - Defines generic CalcNode structures and implements expression evaluation through the `resolve_internal()` method

### **Value Resolution (where overflow occurs):**
3. **`servo/components/style/values/computed/length_percentage.rs`** - Implements `CalcLengthPercentage::resolve()` which evaluates the calc tree and applies clamping, but doesn't validate for infinity/NaN results from overflow

### **C++ FFI and Usage:**
4. **`servo/ports/geckolib/glue.rs`** - `Servo_ResolveCalcLengthPercentage()` FFI function bridges Rust to C++
5. **`layout/style/ServoStyleConstsInlines.h`** - C++ methods `ResolveToCSSPixels()` and `Resolve()` that use the resolved values without overflow checking
6. **`layout/style/nsStyleStruct.cpp`** - C++ code that manipulates calc nodes

### **Root Cause:**
Extreme CSS calc() values (like `calc(999999999999px * 999999999999)`) produce infinity during f32 arithmetic operations in Rust. These infinity values pass unchecked through the FFI boundary into C++ code, where they're used in further calculations causing crashes.
