## Key Discoveries

### Files Analyzed
I examined the Apache Kafka codebase (at commit 0753c489) and found:

1. **New KafkaConsumer API Implementation** - The current production API located in:
   - `clients/src/main/java/org/apache/kafka/clients/consumer/KafkaConsumer.java`
   - `clients/src/main/java/org/apache/kafka/clients/consumer/Consumer.java` (interface)

2. **Internal Implementations**:
   - `AsyncKafkaConsumer` - For new consumer group protocol (KIP-848)
   - `ClassicKafkaConsumer` - For legacy consumer group protocol
   - Both are now marked as @Deprecated

3. **Core Offset Management Components**:
   - `OffsetAndMetadata` - Encapsulates offsets with optional metadata and leader epoch
   - `CommitRequestManager` - Handles offset commit operations
   - `OffsetFetcher` - Manages offset fetching and position reset
   - `SubscriptionState` - Tracks partition positions and states

### Key Behavioral Differences

**New API vs Legacy ConsumerConnector:**

1. **Offset Representation**: Uses explicit `OffsetAndMetadata` objects containing:
   - The offset value
   - Optional metadata (String)
   - Leader epoch for log truncation detection

2. **Position Management**: Two distinct concepts:
   - **Position**: Offset of the next record to be returned (tracked in memory)
   - **Committed**: Last durably stored offset (persisted to broker)

3. **Commit Operations**:
   - `commitSync()` / `commitAsync()` for manual control
   - `auto.commit.interval.ms` for automatic commits
   - Callback-based error handling

4. **Architecture**: Delegation pattern allows dual protocol support while maintaining a single public API
