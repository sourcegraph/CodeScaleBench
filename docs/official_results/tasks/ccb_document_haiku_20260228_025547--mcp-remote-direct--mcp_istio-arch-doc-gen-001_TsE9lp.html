# Istio Pilot's Discovery Service Architecture

## Overview

Istio's **Pilot Discovery Service** is the core control plane component that implements Envoy's xDS (Extensible Discovery Service) protocol, translating Istio's high-level networking configuration into concrete, proxy-ready Envoy resources. This document details the architecture of this subsystem, focusing on how configuration flows from Kubernetes API servers and Istio resources through Pilot to connected Envoy proxies.

At its core, the Discovery Service answers three fundamental questions for every connected Envoy proxy:
1. **What services exist** and where are their endpoints? (EDS/CDS)
2. **How should traffic route** to those services? (RDS/LDS)
3. **What security policies apply** to this traffic? (SDS/RBAC)

The Discovery Service does this by maintaining a push-based streaming connection with each proxy, broadcasting updates whenever the mesh configuration changes.
