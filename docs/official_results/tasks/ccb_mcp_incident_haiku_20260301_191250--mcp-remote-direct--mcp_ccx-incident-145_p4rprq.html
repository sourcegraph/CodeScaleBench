## Summary

I found the Go source files in grafana/grafana that implement the alerting evaluation scheduler tick and detect when evaluation cycles overrun their time budget:

### Key Files:
1. **`pkg/services/ngalert/schedule/schedule.go`** - Main scheduler that processes ticks and logs the warning
2. **`pkg/services/ngalert/schedule/alert_rule.go`** - Rule evaluation routine that detects overrun
3. **`pkg/services/ngalert/schedule/ticker/ticker.go`** - Ticker that emits regular ticks
4. **`pkg/services/ngalert/eval/eval.go`** - Applies evaluation timeout

### How It Works:
The scheduler runs a periodic loop via `schedulePeriodic()` that receives ticks from the ticker. For each tick, `processTick()` schedules alert rules for evaluation. The `runJobFn()` method sends evaluation signals to rule routines via the `Eval()` method.

The key timeout detection happens in `alert_rule.Eval()` (line 206): it performs a non-blocking read of the evaluation channel. If there's already a pending evaluation (the rule hasn't finished processing the previous tick), this pending evaluation is captured as `droppedMsg`. When this happens, `runJobFn()` logs the warning: **"Tick dropped because alert rule evaluation is too slow"** (line 442).

The evaluation timeout budget is enforced in `eval.EvaluateRaw()` which uses `context.WithTimeout()` with the configured `EvaluationTimeout` to create a deadline for the evaluation execution.
