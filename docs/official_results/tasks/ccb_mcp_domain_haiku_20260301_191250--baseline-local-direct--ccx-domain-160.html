## Summary

The bytecode interpreter dispatch loop in OpenJDK is implemented through a sophisticated template-based system. Here are the key findings:

### **Core Components:**

1. **DispatchTable & EntryPoint** - Data structures that map bytecode values to handler addresses, indexed by CPU stack top state
2. **TemplateInterpreter** - Main class managing dispatch tables and entry points  
3. **TemplateInterpreterGenerator** - Generates all interpreter code at startup via `generate_all()`
4. **TemplateTable** - Defines bytecode templates with properties and code generators
5. **InterpreterMacroAssembler** - Architecture-specific dispatch mechanisms

### **Key Files Identified:**

- **Platform-independent**: `templateInterpreter.cpp/hpp`, `templateInterpreterGenerator.cpp/hpp`, `templateTable.cpp/hpp`
- **Architecture-specific implementations** for x86, aarch64, arm, ppc, riscv, s390:
  - `interp_masm_*.cpp/hpp` - Dispatch prolog/epilog logic
  - `templateInterpreterGenerator_*.cpp` - Code generation
  - `templateTable_*.cpp/hpp` - Bytecode implementations

### **Critical Functions:**

- `set_entry_points_for_all_bytes()` - Populates dispatch tables
- `generate_and_dispatch()` - Core bytecode execution and dispatch
- `dispatch_next/dispatch_base/dispatch_epilog()` - Actual dispatch mechanism loading next bytecode and jumping to handler
