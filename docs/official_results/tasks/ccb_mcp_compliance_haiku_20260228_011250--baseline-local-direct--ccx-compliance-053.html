## Summary of Findings

The authorization audit logging system in Kafka spans multiple modules:

### **Primary Audit Logging Implementation:**
- **`metadata/src/main/java/org/apache/kafka/metadata/authorizer/StandardAuthorizerData.java`** - This is the core class that actually produces the audit log entries. The `authorize()` method calls `logAuditMessage()` which logs authorization decisions:
  - ALLOWED events are logged at DEBUG level when `logIfAllowed` is true
  - DENIED events are logged at INFO level when `logIfDenied` is true
  - Uses the logger "kafka.authorizer.logger"

- **`metadata/src/main/java/org/apache/kafka/metadata/authorizer/StandardAuthorizer.java`** - Implements the Authorizer interface and delegates to StandardAuthorizerData, also records metrics for authorization decisions

### **Supporting Files in Specified Directories:**

**`core/src/main/scala/` files:**
- `AuthHelper.scala` - Helper class that calls authorize() with logIfAllowed/logIfDenied control flags
- `KafkaApis.scala` - Broker request handler that calls authorize() for all authorization checks
- `ControllerApis.scala` - Controller request handler that calls authorize() for controller operations
- `AclApis.scala` - Handles ACL management requests

**`clients/src/main/java/` files:**
- `Action.java` - Contains logIfAllowed and logIfDenied flags that control whether decisions are logged
- `AuthorizationResult.java` - Enum with ALLOWED and DENIED values
- `Authorizer.java` - Interface defining the authorization contract
