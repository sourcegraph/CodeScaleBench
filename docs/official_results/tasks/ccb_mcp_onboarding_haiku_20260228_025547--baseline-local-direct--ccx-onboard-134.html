2. **BrowserMainLoop** (`content/browser/browser_main_loop.h/cc`)
   - Main browser process event loop
   - Initializes core browser subsystems (UI thread, IO thread, media services)

3. **RenderThreadImpl** (`content/renderer/render_thread_impl.h/cc`)
   - Main thread of renderer process
   - Routes IPC messages to RenderFrameImpl instances
   - Implements the `mojom::Renderer` interface

4. **ChildProcessLauncher** (`content/browser/child_process_launcher.h/cc`)
   - Asynchronously spawns child processes
   - Manages process lifecycle and priorities
   - Handles Mojo invitation setup for IPC

5. **Mojo Interface** (`content/common/renderer.mojom`)
   - Defines `content.mojom.Renderer` interface
   - Primary communication contract between browser and renderer
   - Maintains FIFO ordering with legacy IPC messages

These five components form the backbone of Chromium's multi-process architecture, working together to maintain separation between processes while enabling efficient communication.
