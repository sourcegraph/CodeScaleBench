  - `pkg/infra/remotecache/remotecache.go` (abstraction layer)
  - `pkg/infra/remotecache/redis_storage.go` (Redis implementation)
  - `pkg/infra/remotecache/memcached_storage.go` (Memcached implementation)
  - `pkg/infra/remotecache/database_storage.go` (Database implementation with GC)
- **Local Cache**: `pkg/infra/localcache/cache.go` (in-memory cache)
- **Supporting files**: Metrics, KV store wrapper, settings, and test fakes

### Architecture:
1. **Cache Key Generation**: SHA256 hash of JSON-encoded query data with datasource UID prefix
2. **Cache Backend Abstraction**: Pluggable CacheStorage interface supporting Redis, Memcached, and Database backends
3. **TTL Policy**: 
   - Database cache: Configurable per-item expiration with 10-minute GC
   - Redis/Memcached: Per-operation expiration support
   - Local cache: 5-minute default with 10-minute cleanup
4. **Query Interception**: Middleware-based approach with request deduplication using singleflight pattern
5. **Metrics**: Prometheus histograms for cache hits/misses and performance tracking

**Important Note**: The OSS version's CachingService is a no-op implementation that always returns cache misses. Actual query result caching and dashboard cache invalidation appear to be implemented in commercial versions or datasource-specific plugins.
