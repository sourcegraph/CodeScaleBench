## Key Findings:

**Entry Point → Instrument::NPV()** traces through:
1. **LazyObject::calculate()** - Lazy evaluation with caching
2. **Instrument::performCalculations()** - Bridges instrument to engine
3. **Two execution paths:**
   - **Analytic:** AnalyticBarrierEngine directly queries YieldTermStructure (rates/dividends) and BlackVolTermStructure (volatility) to compute closed-form price
   - **Monte Carlo:** MCBarrierEngine spawns paths via PathGenerator, prices via BarrierPathPricer

**Path Generation Chain:**
- PathGenerator<GSG> uses GeneralizedBlackScholesProcess
- For each timestep: queries YieldTermStructure (drift) and BlackVolTermStructure (diffusion)
- Applies StochasticProcess::evolve() with discretization scheme
- Returns Path with asset values at each TimeGrid point

**Path Pricing:**
- BarrierPathPricer monitors barrier crossings with Brownian bridge correction
- Calculates payoff and applies discount factors
- MonteCarloModel accumulates samples with variance reduction (antithetic/control variates)

**Design Patterns:**
- LazyObject (deferred calculation + caching)
- Strategy (pluggable engines)
- Template Method (stochastic process evolution)
- Observable/Observer (market data dependency propagation)
- Traits (compile-time polymorphism for MC dimensions)
