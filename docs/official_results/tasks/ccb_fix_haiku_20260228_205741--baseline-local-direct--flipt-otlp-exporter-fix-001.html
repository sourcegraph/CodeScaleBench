# Add OTLP Exporter Support for Tracing

**Repository:** flipt-io/flipt
**Language:** Go
**Difficulty:** hard

## Problem

Flipt currently only supports Jaeger and Zipkin as tracing exporters, limiting observability integration options for teams using OpenTelemetry collectors or other OTLP-compatible backends. Users cannot export trace data using the OpenTelemetry Protocol (OTLP), which is becoming the standard for telemetry data exchange in cloud-native environments. This forces teams to either use intermediate conversion tools or stick with legacy tracing backends.

## Reproduction

1. Enable tracing in Flipt configuration
2. Attempt to set `tracing.exporter: otlp` in configuration
3. Start Flipt service and observe configuration validation errors
4. Note that only `jaeger` and `zipkin` are accepted as valid exporter values

## Key Components

- Tracing exporter initialization — where Jaeger/Zipkin exporters are created
- Configuration schema — where valid exporter values are defined and validated
- Exporter factory/selection logic

## Task

1. Add `otlp` as a supported tracing exporter alongside `jaeger` and `zipkin`
2. When `otlp` is selected, allow configuration of an endpoint (default: `localhost:4317`)
3. Maintain `jaeger` as the default exporter when no exporter is specified
4. Ensure configuration validation accepts all three exporter values without errors
5. Run existing tests to ensure no regressions

## Success Criteria

- Setting `tracing.exporter: otlp` starts the service without validation errors
- OTLP exporter connects to the configured endpoint (or default `localhost:4317`)
- Default behavior (Jaeger) is unchanged when exporter is not specified
- Existing Jaeger and Zipkin configurations continue to work
- All existing tests pass

Excellent! I've successfully implemented OTLP exporter support for Flipt's tracing system. Here's a summary of what was done:

## Implementation Summary

### Changes Made

**1. Configuration Layer** (`internal/config/tracing.go`)
- Added `TracingExporter` type with three constants:
  - `TracingJaeger` (default)
  - `TracingZipkin`
  - `TracingOTLP`
- Extended `TracingConfig` struct with:
  - `Exporter` field for selecting the exporter type
  - `Endpoint` field for OTLP configuration (defaults to `localhost:4317`)
- Updated `setDefaults()` to set Jaeger as default exporter

**2. Exporter Implementation** (`internal/otel/traces/tracing.go`)
- Modified `GetExporter()` to accept `*config.TracingConfig` parameter
- Implemented switch logic supporting three exporters:
  - **Jaeger**: Uses `autoexport.NewSpanExporter` (preserves existing behavior)
  - **Zipkin**: Uses `autoexport.NewSpanExporter` (preserves existing behavior)
  - **OTLP**: Uses `otlptracegrpc.New()` with optional endpoint configuration
- Returns error for unsupported exporter types

**3. Integration** (`internal/cmd/grpc.go`)
- Updated `GetExporter()` call to pass the tracing configuration: `tracing.GetExporter(ctx, &cfg.Tracing)`

**4. Test Coverage** (`internal/otel/traces/tracing_test.go`)
- Created comprehensive test suite with:
  - Test for Jaeger exporter
  - Test for Zipkin exporter  
  - Test for OTLP with custom endpoint
  - Test for OTLP with default endpoint
  - Test for unsupported exporter validation

### Configuration Usage

Users can now configure OTLP exporter in their config files:

```yaml
tracing:
  enabled: true
  exporter: otlp
  endpoint: localhost:4317  # optional, defaults to localhost:4317
```

Or use Jaeger (default behavior):
```yaml
tracing:
  enabled: true
```

func TestGetExporter(t *testing.T) {
	tests := []struct {
		name    string
		cfg     *config.TracingConfig
		wantErr error
	}{
		{
			name: "jaeger exporter",
			cfg: &config.TracingConfig{
				Exporter: config.TracingJaeger,
			},
		},
		{
			name: "zipkin exporter",
			cfg: &config.TracingConfig{
				Exporter: config.TracingZipkin,
			},
		},
		{
			name: "otlp exporter",
			cfg: &config.TracingConfig{
				Exporter: config.TracingOTLP,
				Endpoint: "localhost:4317",
			},
		},
		{
			name: "otlp exporter with default endpoint",
			cfg: &config.TracingConfig{
				Exporter: config.TracingOTLP,
			},
		},
		{
			name: "unsupported exporter",
			cfg: &config.TracingConfig{
				Exporter: "unsupported",
			},
			wantErr: errors.New("unsupported tracing exporter: unsupported"),
		},
	}

	for _, tt := range tests {
		t.Run(tt.name, func(t *testing.T) {
			// Reset the singleton state for each test
			traceExpOnce = sync.Once{}
			traceExp = nil
			traceExpFunc = func(context.Context) error { return nil }
			traceExpErr = nil

			exp, expFunc, err := GetExporter(t.Context(), tt.cfg)

				require.EqualError(t, err, tt.wantErr.Error())
				assert.Nil(t, exp, "expected nil exporter when error occurs")
				return
			}

			require.NoError(t, err)
			assert.NotNil(t, exp)
			assert.NotNil(t, expFunc)

			t.Cleanup(func() {
				err := expFunc(context.Background()) // nolint
				assert.NoError(t, err)
