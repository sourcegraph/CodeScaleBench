## Key Findings

### **PassManager Template** (`llvm/include/llvm/IR/PassManager.h`)
- Core template class that manages pass execution for any IR unit (Module, Function, Loop)
- Stores passes in a vector and executes them sequentially
- `addPass()` method registers passes using type erasure via PassModel
- Two main aliases: `ModulePassManager` and `FunctionPassManager`

### **PassRegistry** (`llvm/include/llvm/PassRegistry.h` and `llvm/lib/IR/PassRegistry.cpp`)
- Singleton registry that maintains all registered passes
- Two maps: by type ID and by command-line argument string
- Passes register themselves via `registerPass()` at program startup

### **Registration Macros** (`llvm/include/llvm/PassSupport.h`)
- `INITIALIZE_PASS`: Generates static registration functions that execute at startup
- `RegisterPass<T>`: Template for direct pass registration
- These create PassInfo objects and store them in the global PassRegistry

### **PassBuilder Pipeline** (`llvm/include/llvm/Passes/PassBuilder.h` and `llvm/lib/Passes/PassBuilderPipelines.cpp`)

  - Module-level passes (inlining, attribute inference)
  - Function-level passes (simplification, loop optimization)
  - Loop-level passes (rotation, deletion, interchange)
  - Vectorization passes (loop vectorization, SLP)
  - Global optimizations (DCE, constant merge, devirtualization)
