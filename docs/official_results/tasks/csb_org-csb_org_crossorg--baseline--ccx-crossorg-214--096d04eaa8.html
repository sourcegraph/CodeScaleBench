# SetupSignalContext Function and Signal Handling

The `SetupSignalContext` function in the Kubernetes apiserver package handles graceful shutdown through OS signals. Here's the implementation:

```go
func SetupSignalContext() context.Context {
    close(onlyOneSignalHandler) // panics when called twice
    
    shutdownHandler = make(chan os.Signal, 2)
    
    ctx, cancel := context.WithCancel(context.Background())
    signal.Notify(shutdownHandler, shutdownSignals...)
    go func() {
        <-shutdownHandler
        cancel()
        <-shutdownHandler
        os.Exit(1) // second signal. Exit directly.
    }()
    
    return ctx
}
```

**Signal Registration**: The function uses `signal.Notify(shutdownHandler, shutdownSignals...)` to listen for SIGTERM and SIGINT signals.

**Shutdown Behavior**: According to the documentation, "a context.Context is returned" and "a second signal is caught, the program is terminated with exit code 1."

**Graceful Termination**: The first signal triggers `cancel()`, initiating graceful shutdown. A second signal forces immediate exit with code 1, preventing indefinite hangs.

**Single-Use Protection**: The `close(onlyOneSignalHandler)` statement ensures this function can only be called once, panicking if invoked multiple times.
