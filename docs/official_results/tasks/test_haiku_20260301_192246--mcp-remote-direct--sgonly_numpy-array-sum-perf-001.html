# Array Sum Optimization Summary

## Overview
Optimized the `numpy.core.multiarray.array_sum` function by replacing the generic ufunc-based reduction approach with `np.einsum` for better performance.

## Location
- **File**: `numpy/_core/_methods.py`
- **Function**: `_sum()`
- **Baseline Runtime**: 0.045 seconds

## Optimization Strategy

### The Problem
The original implementation uses `umr_sum` (which is `um.add.reduce`), a general-purpose ufunc reduction mechanism that works through:
1. Generic reduction framework setup
2. Loop creation and scheduling
3. Element-wise addition operations

### The Solution
For common use cases, leverage NumPy's optimized `einsum` (Einstein summation) implementation, which:
1. Has specialized, heavily optimized C implementations for various tensor contraction patterns
2. Avoids the overhead of the generic reduction framework
3. Uses direct SIMD instructions for summation operations when possible

### Optimization Scope

The einsum-based fast path is activated when:
- `where is True` (no masking)
- `out is None` (no output array)
- `initial is _NoValue` (no initial value)

These constraints ensure the optimization only applies to simple, standard summation cases.

### Implementation Details

The optimization handles multiple axis configurations by constructing appropriate einsum strings:

1. **`axis=None`**: Sum all elements
