## Key Components Documented:

### 1. **Type Class Hierarchy** (IR Layer)
- **Type.h**: Base `Type` class with immutable singleton pattern
- **DerivedTypes.h**: `IntegerType`, `FunctionType`, and `StructType` implementations
- All types are created via static factory methods and uniqued per context

### 2. **Context Ownership** (Management Layer)
- **LLVMContext**: Manages type uniquing and ownership
- **LLVMContextImpl**: Contains type uniquing tables:
  - `IntegerTypes` map
  - `FunctionTypeSet` (FoldingSet)
  - `AnonStructTypes` and `NamedStructTypes`
  - `ArrayTypes`, `VectorTypes`, `PointerTypes` maps

### 3. **IR to Machine Type Conversion** (Code Generation Bridge)
- **ValueTypes.h/cpp**: `EVT` class that wraps both simple (`MVT`) and extended types
- `EVT::getEVT()`: Converts IR `Type*` to `EVT`
- `EVT::getTypeForEVT()`: Reverse conversion
- Extended types maintain pointers back to original IR types

### 4. **SelectionDAG Type Legalization** (Code Generation Layer)
- **LegalizeTypes.h**: `DAGTypeLegalizer` class managing type transformations
- Maintains transformation maps for:
  - Integer promotion/expansion
  - Float softening/promotion
  - Vector scalarization/splitting/widening
- **TargetLowering.h**: `ValueTypeActionImpl` stores target-specific type actions
- **TargetLoweringBase.cpp**: `computeRegisterProperties()` configures type actions
