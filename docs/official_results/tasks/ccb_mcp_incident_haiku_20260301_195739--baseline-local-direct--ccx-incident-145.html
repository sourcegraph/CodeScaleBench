### Detection Mechanism:

- Receives ticks from the ticker and measures latency with `BehindSeconds` metric
- Tracks iteration duration with `SchedulePeriodicDuration` metric

**Evaluation Cycle Overrun Detection** (`runJobFn`, line 438):
- When sending an evaluation to a rule, the scheduler checks if a previous evaluation is still pending
- If a new tick arrives before the previous evaluation completes, the method detects it and logs:
  ```
  "Tick dropped because alert rule evaluation is too slow"
  ```
- Increments `EvaluationMissed` counter to track missed evaluation cycles

**Detection Implementation** (`alertRule.Eval`, lines 200-217):
- Uses non-blocking channel read to detect if an evaluation message is already waiting
- Returns the dropped message to the scheduler, signaling an overrun
