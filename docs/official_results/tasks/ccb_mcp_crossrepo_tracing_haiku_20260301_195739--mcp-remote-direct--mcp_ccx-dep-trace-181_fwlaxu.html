## Key Components in the Fetch Chain

3. **FetchRequest.java** - Request builder for constructing fetch requests
4. **FetchResponse.java** - Response parsing from brokers
5. **FetchSessionHandler.java** - Manages incremental fetch sessions
6. **CompletedFetch.java** - Represents fetched partition data with record parsing
7. **FetchCollector.java** - Orchestrates conversion of CompletedFetch to ConsumerRecords
8. **ConsumerRecords.java** - Final consumer-facing records collection

## Processing Flow

1. **Request Building**: `Fetcher.sendFetches()` calls `AbstractFetch.prepareFetchRequests()` to determine which partitions need fetching
2. **Request Creation**: `AbstractFetch.createFetchRequest()` builds a `FetchRequest` using `FetchRequest.Builder.forConsumer()`
3. **Network Send**: Request is sent through ConsumerNetworkClient
4. **Response Handling**: `AbstractFetch.handleFetchSuccess()` receives the `FetchResponse` and creates `CompletedFetch` objects
5. **Buffering**: CompletedFetch objects are added to FetchBuffer
6. **Record Parsing**: `FetchCollector.collectFetch()` processes buffered fetches, calling `CompletedFetch.fetchRecords()`
7. **Deserialization**: Individual records are parsed by `CompletedFetch.parseRecord()`, which deserializes keys and values
8. **Final Collection**: Parsed `ConsumerRecord` objects are aggregated into `ConsumerRecords` and returned to the user
