## Complete Alerting Pipeline

**1. Rule Scheduler** (`schedule/schedule.go`)
   - `Run()` → `schedulePeriodic()` → `processTick()` → `runJobFn()`
   - Manages periodic evaluation of alert rules based on configured intervals
   - Creates and manages rule evaluation routines

**2. Alert Rule Evaluation** (`schedule/alert_rule.go`)
   - `Run()` - Main loop listening on evaluation channel
   - `Eval()` - Signals rule to evaluate
   - `evaluate()` - Executes evaluation with retry logic and state updates

**3. Condition Evaluation** (`eval/eval.go`)
   - `Evaluate()` - Executes the query/expression pipeline
   - `EvaluateAlert()` - Converts query results to alert instances
   - Handles datasource queries and transformations

**4. State Transition Logic** (`state/manager.go`)
   - `ProcessEvalResults()` - Core function that:
     - Computes state transitions from evaluation results
     - Detects new alerts, resolved alerts, and state changes
     - Persists state to database
     - Determines which states need notification
   - `setNextStateForRule()` - Calculates next state for each alert instance
   - `updateLastSentAt()` - Filters transitions needing to be sent

**5. Notification Dispatch** (`sender/router.go`)
   - `AlertsRouter.Send()` - Routes alerts based on organization configuration:
     - To internal notifier (Prometheus Alertmanager)
     - To external Alertmanager(s)
     - Or both depending on admin configuration
