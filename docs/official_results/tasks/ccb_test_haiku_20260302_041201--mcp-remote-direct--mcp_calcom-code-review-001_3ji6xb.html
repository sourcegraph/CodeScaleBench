You are reviewing a recently merged pull request that adds scope configuration for the feature opt-in system. ## Summary of Findings

### Critical Functional Bugs (3)
All three mutation endpoints in the tRPC router have the same bug:

1. **setUserState (line 103)** - Always passes `assignedBy` even when state is "inherit"
2. **setTeamState (line 131)** - Always passes `assignedBy` even when state is "inherit"  
3. **setOrganizationState (line 161)** - Always passes `assignedBy` even when state is "inherit"

**Root Cause:** The service method signatures use TypeScript union types that explicitly forbid `assignedBy` when state is "inherit". Unconditionally passing it violates the contract.

### High-Severity Compliance Violations (3)

4. **setUserState (line 96)** - Missing scope validation for "user" level
5. **setTeamState (line 124)** - Missing scope validation for "team" level
6. **setOrganizationState (line 153)** - Missing scope validation for "org" level

**Root Cause:** The router validates that slugs exist in the opt-in feature allowlist but doesn't use `isFeatureAllowedForScope()` to validate the correct scope. This deviates from the established pattern in the config module and relies on the service layer for validation instead of performing early validation.
