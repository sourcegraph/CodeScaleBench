### **Configuration Phase (Compile-Time):**
1. **HttpConnectionManager proto** defines `http_filters` field with repeated HttpFilter messages
2. **HttpConnectionManagerFilterConfigFactory** reads the proto config
3. **HttpConnectionManagerConfig** (which implements FilterChainFactory) uses **FilterChainHelper** to process each filter from the config
4. **FilterChainHelper** looks up filter factories in the registry and creates FilterConfigProviders
5. Filter factories are cached in `filter_factories_` list

### **Filter Chain Creation (Runtime):**
1. When a new stream arrives, **createFilterChain()** is called
2. **FilterChainUtility::createFilterChainForFactories()** applies cached factories to the **FilterChainManager**
3. Filter factories invoke **FilterChainFactoryCallbacks** to register actual filter instances

### **Filter Execution (Runtime):**
1. **FilterManager** maintains decoder and encoder filter lists
2. Stream events (headers, data, trailers) trigger sequential iteration through filter chains
3. Filter return codes control continuation vs. buffering
4. Finally reaches terminal filter (router) which generates response
5. Response flows backward through encoder filter chain
