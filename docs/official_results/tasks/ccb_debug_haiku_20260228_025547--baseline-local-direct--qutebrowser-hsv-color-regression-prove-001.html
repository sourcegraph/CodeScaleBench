# Bug Investigation: HSV Color Percentage Parsing Produces Wrong Hue Values

**Repository:** qutebrowser/qutebrowser
**Task Type:** Find and Prove (write a regression test)

## Reported Issue

When users specify colors in the qutebrowser configuration using HSV (Hue-Saturation-Value) notation with percentage values, the hue component is calculated incorrectly.

The hue channel in HSV color space has a valid range of 0-359 degrees. However, the color parser incorrectly scales hue percentages using a maximum of 255 instead of 359. This means:

- `hsv(100%, 100%, 100%)` is parsed as `(255, 255, 255)` instead of the correct `(359, 255, 255)`
- `hsv(50%, 50%, 50%)` produces hue 127 instead of the correct hue 179

The saturation and value channels (which correctly use 0-255 range) are not affected — only the hue channel has this scaling error.

The same bug affects HSVA notation (HSV with alpha channel). For example, `hsva(100%, 100%, 100%, 100%)` also produces the wrong hue value.

# HSV Color Percentage Parsing Bug Analysis

## Issue Summary

When users specify colors in qutebrowser configuration using HSV (Hue-Saturation-Value) notation with percentage values, the hue component is calculated incorrectly.

The hue channel in HSV color space should have a valid range of 0-359 degrees. However, the color parser incorrectly scales hue percentages using a maximum of 255 instead of 359, producing wrong hue values.

## Examples of the Bug

```
Current (Buggy) Behavior:
- hsv(100%, 100%, 100%) → H=254, S=254, V=254  (should be H=359, S=255, V=255)
- hsv(50%, 50%, 50%)    → H=127, S=127, V=127  (should be H=179, S=128, V=128)
- hsv(10%, 10%, 10%)    → H=25,  S=25,  V=25   (should be H=36,  S=25,  V=25)
- hsva(10%, 20%, 30%, 40%) → H=25, S=51, V=76, A=102 (H should be 36, others correct)
```

## Root Cause

**File:** `/app/qutebrowser/config/configtypes.py` (lines 1004-1018)

**Method:** `QtColor._parse_value()`

The bug is in the `_parse_value()` method which parses color component values. When it encounters a percentage value, it applies a uniform scaling factor:

```python
def _parse_value(self, val: str) -> int:
    try:
        return int(val)
    except ValueError:
        pass

    mult = 255.0
    if val.endswith('%'):
        val = val[:-1]
        mult = 255.0 / 100  # ← BUG: Uses 255 for all components

    try:
        return int(float(val) * mult)
    except ValueError:
        raise configexc.ValidationError(val, "must be a valid color value")
```

**The Problem:**
- For HSV colors, the hue component ranges from 0-359 degrees
- Saturation and value components range from 0-255
- The `_parse_value()` method doesn't distinguish between these components
- It always scales percentages using `255 / 100`, which is correct for S/V but wrong for H
