# Kubernetes Scheduler Architecture

## Core Files & Responsibilities

**Scheduler Core** (pkg/scheduler/):
- `scheduler.go`: Main Scheduler struct managing Cache, SchedulingQueue, Profiles
- `schedule_one.go`: Two-phase pipeline - ScheduleOne → schedulingCycle → bindingCycle (async)
- `eventhandlers.go`: Pod/Node event handlers populating queue

**Framework** (pkg/scheduler/framework/):
- `interface.go`: Plugin interfaces, Status codes, Extension points
- `cycle_state.go`: Thread-safe key-value store for plugin data sharing
- `runtime/framework.go`: Plugin execution orchestration

**Queue & Cache** (pkg/scheduler/internal/):
- `queue/scheduling_queue.go`: 3-queue system (activeQ, backoffQ, unschedulablePods)
- `cache/cache.go`: Node snapshots + assumed pod tracking

## Key Architectural Patterns

1. **Two-Phase Pipeline**: Synchronous scheduling cycle (fast, ~ms) + asynchronous binding cycle (slower, ~s)
   - Scheduling: filter nodes → score nodes → reserve resources
   - Binding: finalize binding via API → post-bind cleanup
   - Separation enables parallel pod scheduling

2. **Snapshot-Based Filtering**: Immutable node state taken once per cycle
   - Avoids race conditions; deterministic behavior
   - Efficient for large clusters (only changed nodes updated)

3. **Assume Pattern**: Optimistic pod placement before binding
   - Adds pod to cache immediately after scheduling
   - Enables subsequent pods to see assumed placement
   - Reverted if binding fails
