## Key Files in the Android Rendering Pipeline

1. **View.java** (core/java/android/view/View.java) - Contains the fundamental rendering methods:

2. **ViewRootImpl.java** (core/java/android/view/ViewRootImpl.java) - Bridges WindowManager and the view hierarchy:
   - `scheduleTraversals()` (line 2948) - Schedules view tree traversals via Choreographer
   - `performTraversals()` (line 3422) - Executes the complete measure→layout→draw pipeline

3. **Choreographer.java** (core/java/android/view/Choreographer.java) - Handles VSYNC synchronization:
   - `doFrame()` - Executes frame callbacks synchronized with VSYNC
   - `scheduleVsync()` - Requests the next VSYNC signal

4. **ThreadedRenderer.java** (core/java/android/view/ThreadedRenderer.java) - Hardware-accelerated rendering on a separate render thread

5. **HardwareRenderer.java** (graphics/java/android/graphics/HardwareRenderer.java) - GPU rendering interface with `syncAndDraw()` method

6. **RecordingCanvas.java** & **BaseRecordingCanvas.java** (graphics/java/android/graphics/) - Records drawing commands for GPU replay

The complete rendering pipeline flow is: VSYNC signal → Choreographer frames → ViewRootImpl scheduling → performTraversals (measure→layout→draw) → ThreadedRenderer with RecordingCanvas → GPU rendering.
