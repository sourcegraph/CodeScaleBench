## Key Findings

**Architecture Pattern**: The Django ORM uses a sophisticated combination of:
1. **Lazy evaluation** - QuerySets defer SQL generation until iteration
2. **Expression tree visitor pattern** - Compilable objects recursively generate SQL
3. **Vendor dispatch** - The `as_{vendor}()` method pattern allows database-specific SQL generation

**Critical Files** (7 core + backend-specific):
- `Manager` → `QuerySet` (high-level API)
- `Query` + `WhereNode` (internal representation)
- `SQLCompiler` (translation to SQL)
- `Lookup` + `Expression` (SQL fragment generation)
- `BaseOperations.compiler()` (vendor routing)

**Data Flow**:
1. `filter()` accumulates conditions in a `Query.where` WhereNode tree
2. On evaluation (iteration/count), a backend-specific `SQLCompiler` is instantiated
3. `as_sql()` recursively compiles the expression tree
4. Vendor dispatch checks for `as_{vendor}()` methods (e.g., `Lookup.as_oracle()`)
5. Final SQL and parameters passed to the database driver
