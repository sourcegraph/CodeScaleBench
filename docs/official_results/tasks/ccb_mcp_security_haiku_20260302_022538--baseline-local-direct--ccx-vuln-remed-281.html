4. **src/spawn_sync.cc** - Synchronous spawn implementation
5. **src/permission/child_process_permission.cc** - Permission system integration

### **Core Security Functions:**

- **`validateArgumentNullCheck()`** (lib/child_process.js:963-967) - The primary validator that prevents null-byte injection by checking for `\u0000` characters
- **`validateArgumentsNullCheck()`** (lib/child_process.js:970-974) - Validates arrays of arguments
- **`normalizeSpawnArguments()`** (lib/child_process.js:538-709) - Central validation point for spawn arguments
- **`normalizeExecArgs()`** & **`normalizeExecFileArgs()`** - Validate exec/execFile specific arguments

### **Execution Chain:**

The security flow follows this path:
1. **spawn()** → normalizes and validates all arguments
2. **normalizeSpawnArguments()** → checks all strings for null bytes
3. **ChildProcess.prototype.spawn()** → passes validated options to C++
4. **ProcessWrap::Spawn()** → native implementation with permission checks

### **What It Prevents:**

- **Null-byte injection** - Prevents early string termination in C APIs
- **Shell command injection** - Validates command strings and arguments
- **Environment variable injection** - Checks both keys and values
- **Path/process name spoofing** - Validates argv0 and shell parameters
