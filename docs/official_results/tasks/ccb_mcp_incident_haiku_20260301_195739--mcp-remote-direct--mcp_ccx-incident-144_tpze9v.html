### Key Decision Logic
The main eviction decision logic flows through:
1. **`synchronize()`** - Main monitoring loop in `eviction_manager.go`
2. **`makeSignalObservations()`** - Collects current memory metrics
3. **`thresholdsMet()`** - Compares available memory against thresholds
4. **`rankMemoryPressure()`** - Ranks pods for eviction priority
5. **`evictPod()`** - Executes the actual pod termination

### Decision Logic Details
The threshold comparison in `thresholdsMet()` uses the `OpLessThan` operator: pods are evicted when available memory falls below the configured threshold value. Memory thresholds trigger `NodeMemoryPressure` condition which is reported via `IsUnderMemoryPressure()`.
