## Summary of Defects Found

All three defects are **type mismatches in the tRPC router** that violate the service interface contract:

### Critical Bugs (3)

1. **setUserState mutation (line 103)**: Always passes `assignedBy` even when state is `"inherit"`, violating the discriminated union type that specifies `assignedBy` should only be present for `"enabled"` or `"disabled"` states.

2. **setTeamState mutation (line 131)**: Same type mismatch - unconditionally passes `assignedBy` regardless of the state value.

3. **setOrganizationState mutation (line 161)**: Same type mismatch affecting the organization settings endpoint.

### Root Cause

The service interface defines a discriminated union for the `setUserFeatureState` and `setTeamFeatureState` methods:
```typescript
input:
  | { ...; state: "enabled" | "disabled"; assignedBy: number }
  | { ...; state: "inherit" }  // NO assignedBy
```

The router violates this contract by always including `assignedBy` in the input object, causing TypeScript compilation errors.

### Impact

- **Severity**: Critical - These are type violations that prevent the code from compiling with strict TypeScript checking
- **Functional Impact**: The service implementation correctly handles the type discrimination, but the router bypasses it, creating a type safety gap
- **Compliance**: Deviates from the established interface contract

All fixes are provided as unified diffs in `/workspace/review.json` and conditionally construct the input object based on state value, ensuring type safety.
