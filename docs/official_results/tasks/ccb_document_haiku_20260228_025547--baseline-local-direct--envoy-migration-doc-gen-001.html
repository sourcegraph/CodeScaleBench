# Envoy External Processing (ext_proc) Filter Migration Guide
## Version 1.30.0 → 1.31.0

## Overview

This guide documents the changes to the Envoy External Processing HTTP filter between v1.30.0 (April 2024) and v1.31.0 (July 2024). The ext_proc filter enables external gRPC services to process HTTP requests and responses in flexible ways.

**Summary of Major Changes:**
- New `observability_mode` feature for monitoring HTTP traffic without blocking
- Enhanced route cache control via new `route_cache_action` enum field
- Support for deferred stream closure in observability mode
- HTTP status code improvement for timeout errors
- Deprecated `async_mode` in favor of `observability_mode`

---

## Breaking Changes

### 1. HTTP Status Code for External Processor Timeouts

**Impact:** Applications expecting `500 Internal Server Error` for ext_proc timeouts will now receive `504 Gateway Timeout`.

**Details:**
- **v1.30.0:** Timeout errors in external processor returned `500 Internal Server Error`
- **v1.31.0:** Timeout errors now return `504 Gateway Timeout`
- This is semantically more correct as it indicates the upstream service (external processor) is not responding

**Migration Action:**
- Update monitoring/alerting rules that depend on HTTP status codes for ext_proc failures
- Adjust client-side retry logic if it differentiates between 5xx error codes
- No configuration changes required

**Source Files:**
- `source/extensions/filters/http/ext_proc/ext_proc.cc` - Filter implementation
