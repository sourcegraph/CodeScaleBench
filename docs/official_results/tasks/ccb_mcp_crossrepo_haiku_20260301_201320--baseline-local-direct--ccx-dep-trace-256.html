**The Four Main Components:**

1. **DIE (Debug Info Entry) Creation** - The `die_struct` (line 3146) is the core data structure. DIEs are created via `new_die_raw()` and `new_die()`, with attributes added via numerous `add_AT_*` functions.

3. **DWARF Output Generation** - The chain flows: front-end hooks → `dwarf2out_decl()` → `gen_decl_die()` → `gen_subprogram_die()`/`gen_type_die()` → `new_die()` with attributes.

4. **Line Number Table Emission** - `dwarf2out_source_line()` records line info during code generation, then `output_line_info()` generates the complete `.debug_line` section with file tables and line number programs.
