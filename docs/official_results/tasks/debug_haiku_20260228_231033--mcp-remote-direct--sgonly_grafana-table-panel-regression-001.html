## Key Discovery

The regression is caused by **incomplete tracking logic** in the migration pipeline that fails to preserve field configuration (overrides) for table panels without explicit `fieldConfig.defaults.custom`.

### Root Cause

**Location:** `apps/dashboard/pkg/migration/frontend_defaults.go`, function `trackPanelOriginalFieldConfigCustom()` (lines 1150-1169)

The tracking function only marks panels that have `fieldConfig.defaults.custom`, but it completely ignores panels with:
- Non-empty `fieldConfig.overrides` (which contain column formatting, alignment, etc.)
- But NO `fieldConfig.defaults.custom`

1. **Tracking Phase (v10.3 dashboards):** Panels with field overrides but no `defaults.custom` don't get marked with `_originallyHadFieldConfigCustom`

- The marker is set to `true`
- The cleanup logic preserves the fieldConfig and all overrides
- No data is lost

**V38 migration** - The issue manifests when upgrading from Grafana v10.3 to v10.4, affecting dashboard imports through schema version 38 or later.
