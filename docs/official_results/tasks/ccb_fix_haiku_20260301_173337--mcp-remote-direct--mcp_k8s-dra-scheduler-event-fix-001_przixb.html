# big-code-k8s-bug-001: Missing ResourceSlice Event Handler in Kubernetes Scheduler

## Task

Investigate a bug in the Kubernetes scheduler where pods requesting Dynamic Resource Allocation (DRA) devices become permanently stuck in an Unschedulable state when the DRA driver starts after the pod is created. Trace the execution path through the scheduler's event-driven re-queuing architecture to identify why ResourceSlice creation/update events are silently dropped.

## Context

- **Repository**: github.com/sg-evals/kubernetes--v1.30.0 (mirror of github.com/sg-evals)/kubernetes--v1.30.0 (mirror of kubernetes/kubernetes) (Go, ~3.5M LOC)
- **Category**: Bug Investigation
- **Difficulty**: hard
- **Entry Point**: `pkg/scheduler/schedule_one.go` — `ScheduleOne()` method

## Symptom

When a pod requests a DRA device and the DRA driver starts *after* the pod is created, the pod becomes permanently stuck in `Pending` with reason `Unschedulable`. The scheduler logs `"cannot allocate all claims"` because no ResourceSlices exist at scheduling time. After the driver starts and publishes ResourceSlice objects, the pod should be rescheduled — but it never is.

The bug is a race condition: if the driver starts first, scheduling succeeds immediately. But if the pod arrives before the driver, the scheduler never retries because ResourceSlice events are not wired into the scheduling queue's re-evaluation mechanism.

## Summary

I've completed a thorough investigation of the Kubernetes scheduler bug where pods requesting Dynamic Resource Allocation (DRA) devices become permanently stuck when the DRA driver starts after the pod is created.

### Key Findings:

**Root Causes Identified:**

1. **Missing GVK Constant** - `pkg/scheduler/framework/types.go` (line 96)
   - `framework.ResourceSlice` is not defined in the GVK constants block

2. **Missing Event Registration** - `pkg/scheduler/framework/plugins/dynamicresources/dynamicresources.go` (line 410)
   - The DynamicResources plugin's `EventsToRegister()` method doesn't include ResourceSlice events

3. **Missing Event Handler** - `pkg/scheduler/eventhandlers.go` (line 492)
   - No case handler for ResourceSlice in the `addAllEventHandlers()` switch statement

### Why This Is a Bug:

The scheduler uses **event-driven re-queuing** to move pods from the unschedulable queue back to active queue when cluster state changes. When a pod requests DRA resources:

1. If ResourceSlices don't exist when the pod arrives → scheduling fails
2. Pod is added to unschedulable queue
3. When ResourceSlices are later created → **no event handler triggers re-queuing**
4. Pod stays stuck in Unschedulable state forever

### Proposed Fix:

Three coordinated changes:

1. Add `ResourceSlice GVK = "ResourceSlice"` constant to `pkg/scheduler/framework/types.go`
2. Register ResourceSlice events in the DynamicResources plugin's `EventsToRegister()` method
3. Add a case handler for ResourceSlice in `pkg/scheduler/eventhandlers.go`
