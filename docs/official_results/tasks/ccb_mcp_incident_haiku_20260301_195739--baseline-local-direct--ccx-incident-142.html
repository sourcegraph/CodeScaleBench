## Key Findings

I identified **three critical Java source files** in Apache Kafka that handle the heartbeat session timeout:

- Reads the configuration from `SESSION_TIMEOUT_MS_CONFIG`

### 2. **Heartbeat.java** - Manages the timeout timer

- Provides the `sessionTimeoutExpired()` method that checks if the timeout has expired
- Resets the timer when heartbeats are successfully received

### 3. **AbstractCoordinator.java** - Raises/logs timeout during polling
- Line 1513: Checks `heartbeat.sessionTimeoutExpired(now)` in the heartbeat thread's polling loop
- Line 1516-1517: When timeout is detected, logs "session timed out without receiving a heartbeat response"
- The `markCoordinatorUnknown()` method (line 1006-1008) logs when the coordinator becomes unavailable due to the timeout
- This causes the consumer to drop group membership and attempt coordinator rediscovery
