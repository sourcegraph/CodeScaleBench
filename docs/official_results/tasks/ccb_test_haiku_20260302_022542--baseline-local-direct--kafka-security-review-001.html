## Security Defects Identified

1. **ScramSaslServer.java (line 135)** - **CRITICAL: Inverted Iteration Count Validation**
   - Bug: Uses `>` instead of `<` in the condition
   - Impact: Rejects credentials with higher iteration counts (more secure) while allowing lower counts (weaker)
   - Attack: Enables use of weak credentials, weakening cryptographic security

2. **CredentialCache.java (line 36)** - **CRITICAL: Removed Credential Type Check**
   - Bug: Type validation logic is commented out
   - Impact: Wrong credential types can be retrieved for a mechanism
   - Attack: Credential confusion attack - type mismatch could cause unsafe casting or bypasses

3. **SaslServerAuthenticator.java (line 698)** - **CRITICAL: Disabled Session Expiration**
   - Bug: Session expiration timestamp calculation is commented out
   - Impact: `sessionExpirationTimeNanos` never gets set, remaining null
   - Attack: Sessions persist indefinitely, bypassing credential expiration enforcement

4. **AclAuthorizer.scala (line 520)** - **CRITICAL: Inverted ACL Logic**
   - Bug: Logic uses `(denyAclExists && allowAclExists)` instead of `(!denyAclExists && allowAclExists)`
   - Impact: Allows access when BOTH deny and allow rules exist
   - Attack: Authorization bypass - deny rules are ineffective when allow rules also exist

All defects are **authentication/authorization bypasses** with unified diff patches provided in the JSON file.
