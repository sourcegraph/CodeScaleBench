## Summary of Key Findings:

The Firefox event dispatch system flows through these main stages:

1. **Event Creation** - `EventDispatcher::CreateEvent()` wraps widget events as DOM events
2. **Event Target Chain Building** - `EventDispatcher::Dispatch()` walks the DOM tree via `GetEventTargetParent()` to build an array of target items
3. **Capture Phase** - Dispatched from root to target through `EventTargetChainItem::HandleEventTargetChain()`
4. **Target Phase** - Listeners at both capture and bubble on the target element fire
5. **Bubble Phase** - Dispatches from target parent back up to root
6. **Event Listener Invocation** - `EventListenerManager::HandleEventSingleListener()` invokes each listener's callback
7. **Default Action Execution** - `ActivationBehavior()` runs element-specific default behaviors (form submission, checkbox toggle, etc.)

The key files identified are:
- **dom/events/EventDispatcher.h/cpp** - Core dispatch logic
- **dom/events/EventListenerManager.h/cpp** - Listener management and invocation  
- **dom/events/EventTarget.h** - Base event target interface
- **widget/BasicEvents.h** - Widget event definitions
- **layout/base/PresShell.cpp** - PresShell integration
- **dom/html/HTMLButtonElement.cpp**, **HTMLInputElement.cpp** - Default action implementations
