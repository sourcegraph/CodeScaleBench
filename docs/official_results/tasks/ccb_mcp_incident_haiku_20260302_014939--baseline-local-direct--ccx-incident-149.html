## Key Findings

### **Parsing Files**

### **Numeric Overflow Handling (Critical for the crash)**
- **gfx/src/nsCoord.h**: Contains the essential overflow protection functions:
  - `NSToCoordRoundWithClamp()` - Clamps extreme floating-point values before converting to nscoord (32-bit signed int)
  - `NSCoordSaturatingMultiply()` - Safely multiplies coordinates
  - `NSCoordSaturatingAdd()` - Safely adds coordinates  
  - `NSCoordSaturatingSubtract()` - Safely subtracts coordinates
  - `NSFloatPixelsToAppUnits()` / `NSDoublePixelsToAppUnits()` - Unit conversion with overflow protection

### **Value Resolution Chain**
- **layout/style/ServoStyleConstsInlines.h**: Contains `StyleCalcLengthPercentage` class with:
  - `ResolveToCSSPixels()` - Calls Servo's Rust calc resolver
  - `Resolve()` - Converts results to app units with overflow protection

- **layout/style/nsComputedDOMStyle.cpp/h**: Handles DOM style computation with negative calc() clamping
- **layout/style/nsStyleStruct.cpp/h**: Resolves calc values for specific CSS properties

### **Root Cause of Crash**
The crash occurs when Firefox processes CSS calc() expressions with extreme values. The fix involves bounds-checking in `NSToCoordRoundWithClamp()` before converting floating-point calc results to the fixed `nscoord` integer type (which has a maximum value of 2^30-1). Without this check, overflow during the cast can cause memory corruption and crashes.
