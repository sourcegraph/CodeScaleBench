
After inode revalidate failure all mounts on this inode are unmounted and so autofs mount disappears, while automount daemon itself continue to run.

tcpdump shows, that server replies to getattr without error, but client doesn't see this reply, and returning error instead.

## Fault Localization Summary

**Buggy File:** `net/sunrpc/socklib.c`  
**Buggy Function:** `xdr_partial_copy_from_skb()`  
**Confidence Level:** 0.95

### The Bug

In the `xdr_partial_copy_from_skb()` function at lines 96-101, there is an **order-of-operations bug** that causes incorrect calculation of the page length (`pglen`) when copying data from socket buffers to XDR buffers.

**Problematic Code:**
```c
if (base || xdr->page_base) {
    pglen -= base;        // LINE 97: Subtracts ORIGINAL base value
    base += xdr->page_base;  // LINE 98: Adjusts base
    ppage += base >> PAGE_CACHE_SHIFT;  // LINE 99: Advances page pointer
    base &= ~PAGE_CACHE_MASK;  // LINE 100: Masks to get offset within page
}
```

### The Issue

Line 97 subtracts the **original `base` value** from `pglen` before `base` is adjusted by adding `xdr->page_base` and masking. This causes `pglen` to have an **incorrect value** that doesn't match the actual offset within the first page of data.

The correct sequence should be:
1. Adjust `base` to the absolute offset (add `page_base`)
2. Calculate which page we start at
3. Mask `base` to get the offset **within** that page
4. **THEN** subtract this adjusted `base` from `pglen`
