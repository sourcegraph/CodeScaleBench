3. **Evaluation Engine** (`eval/eval.go`): Executes rule queries/expressions using the expression service to produce metric results

4. **State Manager** (`state/manager.go`): Processes evaluation results, determines state transitions, and decides which alerts to send based on thresholds and timing

5. **Alerts Router** (`sender/router.go`): Routes alerts to internal Alertmanager and/or external Alertmanagers based on organization configuration

6. **Multi-Org Alertmanager** (`notifier/multiorg_alertmanager.go`): Maintains per-organization Alertmanager instances for isolation

7. **Internal Alertmanager** (`notifier/alertmanager.go`): Receives alerts and dispatches them through the notification routing engine to receivers/integrations
