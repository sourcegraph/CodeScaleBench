## Processing Chain Overview

- `GrpcMuxImpl::onDiscoveryResponse()` is the entry point for all DiscoveryResponse messages
- Extracts type_url, validates subscriptions, and routes to subscription handlers
- Tracks control plane identifier and manages response nonce

**3. Resource Decoding & Validation:**

- Type URL validation ensures resources match the subscription type
- Custom validators execute via `config_validators_->executeValidators()`

- `SotwSubscriptionState` tracks `last_good_version_info_` and `last_good_nonce_` for ACKs
- `DeltaSubscriptionState` maintains per-resource version maps
- `UpdateAck` structures contain nonce and error details for next request

- **Listeners:** `LdsApiImpl::onConfigUpdate()` → `ListenerManager::addOrUpdateListener()`
- **Clusters:** `CdsApiImpl::onConfigUpdate()` → `CdsApiHelper` → `ClusterManager`
