## Key Findings

### Three Core Components:

1. **Reconciler Interface** (`vendor/sigs.k8s.io/controller-runtime/pkg/reconcile/reconcile.go`)
   - Single method: `Reconcile(context.Context, Request) (Result, error)`
   - Core abstraction that all controllers must implement

2. **Manager Interface** (`vendor/sigs.k8s.io/controller-runtime/pkg/manager/manager.go`)
   - Initializes shared dependencies (caches, clients)
   - Manages controller lifecycle
   - Created via `manager.New(config, options)`

3. **Builder Pattern** (`vendor/sigs.k8s.io/controller-runtime/pkg/builder/controller.go`)
   - Fluent API for configuring controllers
   - Methods: `.For()`, `.Owns()`, `.Watches()`, `.WithEventFilter()`, `.Named()`
   - Completion methods: `.Complete(reconciler)` or `.Build(reconciler)`

### Registration Flow:

1. Create Manager: `mgr := manager.New(config, options)`
2. Create Builder: `builder := ControllerManagedBy(mgr)`
3. Configure: `builder.For(&MyResource{}).Owns(&Pod{})`
4. Register: `builder.Complete(reconciler)`
5. Start: `mgr.Start(ctx)`

### SetupWithManager Pattern:
Controllers encapsulate their registration logic:
```go
func (r *MyReconciler) SetupWithManager(mgr ctrl.Manager) error {
    return ctrl.NewControllerManagedBy(mgr).
        For(&myv1.MyResource{}).
        Complete(r)
}
```
