**3. Frame Style Recomputation** - nsIFrame (layout/generic/nsIFrame.cpp) receives and caches the computed styles

**4. Dirty-Frame Reflow Scheduling** - PresShell::FrameNeedsReflow (layout/base/PresShell.cpp:2632) marks frames as dirty and schedules them for layout recalculation

**5. Paint Triggering** - PresShell::WillPaint (layout/base/PresShell.cpp:10134) flushes pending layout, and PresShell::PaintInternal initiates painting

**6. Display List Construction** - nsLayoutUtils::PaintFrame (layout/base/nsLayoutUtils.cpp:2834) builds a display list by:
   - Creating a RetainedDisplayListBuilder (layout/painting/RetainedDisplayListBuilder.cpp)
   - Calling nsIFrame::BuildDisplayListForStackingContext recursively to create display items
   - Storing items in nsDisplayList (layout/painting/nsDisplayList.cpp)

**7. Display List Rendering** - nsDisplayList::PaintRoot (layout/painting/nsDisplayList.cpp:2348) processes the display list for rendering

**8. WebRender Update** - WebRenderLayerManager (gfx/layers/wr/WebRenderLayerManager.cpp) receives the display list

**9. WebRender Command Generation** - WebRenderCommandBuilder::BuildWebRenderCommands (gfx/layers/wr/WebRenderCommandBuilder.cpp) converts Gecko display items into WebRender primitives and sends them via IPC to the renderer process
