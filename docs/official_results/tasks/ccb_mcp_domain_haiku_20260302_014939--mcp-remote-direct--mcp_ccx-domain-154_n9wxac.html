## Summary

The HTTP filter chain extension architecture in Envoy is built on three core components:

### **1. Core Infrastructure Files:**
- **envoy/http/filter_factory.h** - Defines `FilterChainManager`, `FilterFactoryCb`, and `FilterChainFactory` interfaces
- **envoy/http/filter.h** - Defines `FilterChainFactoryCallbacks`, `StreamFilter`, `StreamDecoderFilter`, `StreamEncoderFilter`

- **source/common/http/filter_manager.h/cc** - Per-request filter chain execution engine with `FilterManager` and `FilterChainFactoryCallbacksImpl`
- **source/common/http/filter_chain_helper.h/cc** - Filter chain utility for converting factory callbacks to active filters
- **source/common/filter/config_discovery_impl.h/cc** - Dynamic filter configuration support via ECDS
- **source/extensions/filters/http/common/factory_base.h** - Common template base for filter implementations

### **Key Design:**
- **Factory Pattern**: Filters are created at configuration time as closures (`FilterFactoryCb`), instantiated at stream time
- **Registry**: Filters register via `REGISTER_FACTORY()` macro for dynamic discovery
- **Chain Execution**: `FilterManager::applyFilterFactoryCb()` executes the filter factory callback, which uses `FilterChainFactoryCallbacks` to add filters to the stream
