   - `TypedMaxOfRateLimiter` (combines multiple limiters)
   - `DefaultTypedControllerRateLimiter` (combines exponential + bucket)

### Controllers Using the Pattern
**25+ controller implementations** that use the work queue:
- Job, namespace, TTL, CronJob controllers
- Endpoint slice, node, route, service controllers
- Certificate, bootstrap, disruption controllers
- And many more across pkg/controller/ and staging/src/k8s.io/

### Key Architectural Pattern
The reconciliation loop implemented across all controllers:
1. `queue.Get()` - retrieves next item to process
2. `syncHandler()` - attempts to reconcile desired vs actual state
3. `queue.Done()` - marks item as processed
4. On success: `queue.Forget()` - stops retry tracking
5. On failure: `queue.AddRateLimited()` - requeues with backoff
