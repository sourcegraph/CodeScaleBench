### Control Flow:

The filter visibility is controlled through a chain of `has_output()` method calls:

1. **ListFilter** (line 26 in filters.py) - Defines abstract `has_output()` method
2. **FieldListFilter** (line 171 in filters.py) - Implements `has_output()` returning `True`

5. **ChangeList.get_queryset()** (line 527 in views/main.py) - Stores only filters with `has_output() == True` in `self.filter_specs`
6. Template iteration (line 72 in change_list.html) - Renders only the filters in `cl.filter_specs`

The critical insight is that **RelatedFieldListFilter.has_output()** prevents empty related-field filters from being displayed by checking if `len(self.lookup_choices) + extra > 1`, where `extra` accounts for the empty choice option.
