### How Overrun Detection Works:

The scheduler detects evaluation cycle overruns through an elegant channel-based mechanism:

1. **Tick Generation**: The scheduler emits periodic ticks at a configured base interval
2. **Evaluation Dispatch** (`runJobFn()` in schedule.go): For each tick, the scheduler attempts to send an evaluation to the rule's evaluation channel
3. **Overrun Detection** (`Eval()` in alert_rule.go): The Eval method performs a non-blocking read on the evaluation channel. If there's already an unprocessed evaluation in the channel, it means the previous evaluation is still running
4. **Warning Logged**: When `dropped != nil`, the warning is logged: **"Tick dropped because alert rule evaluation is too slow"** and the metric `grafana_alerting_schedule_rule_evaluations_missed_total` is incremented

The overrun is detected by the fact that a new evaluation arrived before the previous one was consumed, indicating the evaluation exceeded its time budget.
