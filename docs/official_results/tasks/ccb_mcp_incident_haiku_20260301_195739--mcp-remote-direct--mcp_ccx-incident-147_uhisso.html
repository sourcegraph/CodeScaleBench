### Key Findings:

**Files Involved:**
1. **tsdb/head_append.go** - Contains sample validation logic
2. **tsdb/head.go** - Contains series creation and lifecycle callbacks
3. **storage/interface.go** - Defines error types

**Core Mechanisms:**

1. **Series Limit Enforcement:** The `SeriesLifecycleCallback` interface (defined in tsdb/head.go) allows enforcing series cardinality limits. The `PreCreation()` method is called in `Head.getOrCreateWithOptionalID()` (line 1845) before creating a new series. External systems can implement this callback to reject series creation when limits are exceeded.

2. **Out-of-Order/Out-of-Bounds Handling:** Three appendable methods in tsdb/head_append.go validate samples:
   - `appendable()` (lines 643-683) - for float samples
   - `appendableHistogram()` (lines 688-725) - for histogram samples  
   - `appendableFloatHistogram()` (lines 730-767) - for float histogram samples

3. **Error Types:** Samples are rejected with specific errors when:
   - `ErrOutOfBounds` - timestamp < minValidTime and OOO is disabled
   - `ErrOutOfOrderSample` - sample is out of order and OOO is disabled
   - `ErrTooOldSample` - sample falls outside the OOO time window
