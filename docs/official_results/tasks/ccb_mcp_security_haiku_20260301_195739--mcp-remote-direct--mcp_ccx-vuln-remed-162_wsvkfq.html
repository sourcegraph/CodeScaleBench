## Summary of Findings

The service account token auto-mounting system in Kubernetes consists of three main components:

### 1. **Admission Controller** (`plugin/pkg/admission/serviceaccount/admission.go`)
- **Plugin class**: The admission controller that mutates pod specs during creation

- **mountServiceAccountToken()**: Injects token volumes into containers

- **startServiceAccountTokenController()**: Initializes the token controller with private key, root CA, and token generator
- **newServiceAccountTokenControllerDescriptor()**: Registers the controller to run first before other controllers

### 4. **Type Definitions**
- Both `PodSpec` and `ServiceAccount` types define the `AutomountServiceAccountToken` field
- The admission controller defaults pods to the "default" service account if unspecified
