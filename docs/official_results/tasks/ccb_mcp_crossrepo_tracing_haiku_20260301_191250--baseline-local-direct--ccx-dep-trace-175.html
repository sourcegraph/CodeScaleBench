## Notification Pipeline Flow

The pipeline consists of these key stages:

1. **Alert Ingestion** (`api/v2/api.go::postAlertsHandler`)
   - HTTP POST endpoint receives alerts, validates them, and stores them

2. **Routing Tree Traversal** (`dispatch/route.go::Route.Match`)
   - Depth-first left-to-right search through configured routing rules
   - Returns all matching routes for each alert's label set

3. **Alert Grouping** (`dispatch/dispatch.go::groupAlert`)
   - Groups alerts into aggregation groups based on route configuration
   - Uses `getGroupLabels()` to determine group membership

4. **Inhibition Check** (`inhibit/inhibit.go::Inhibitor.Mutes`)
   - Checks if alerts are inhibited by higher-level source alerts
   - Applies inhibition rules based on label matching

5. **Silence Check** (`silence/silence.go::Silencer.Mutes`)
   - Checks if alerts are silenced
   - Queries active and pending silences for label matches

6. **Final Notifier Dispatch** (`notify/notify.go::Integration.Notify`)
   - Routes alerts to configured receivers
   - Retries with exponential backoff via `RetryStage`
   - Sends to actual integrations (Slack, PagerDuty, email, etc.)

## Key Files
- **dispatch/** - routing and alert grouping
- **notify/** - notification pipeline stages
- **inhibit/** - inhibition rules
- **silence/** - silence management
- **api/v2/** - HTTP endpoints for alert ingestion
