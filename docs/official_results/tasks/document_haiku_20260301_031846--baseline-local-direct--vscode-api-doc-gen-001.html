# VS Code Diagnostic API Reference

## Overview

The Diagnostic API allows VS Code extensions to report code issues (errors, warnings, information, hints) to users within the editor. Diagnostics are displayed in the editor as squiggly lines, in the Problems panel, and in the breadcrumb. This API is essential for linters, language servers, and any extension that needs to provide feedback about code quality or correctness.

The Diagnostic API is exposed through the `vscode.languages` namespace and provides:
- A way to create and manage diagnostics collections
- Severity levels to categorize issue importance
- Support for related diagnostic information and tags for additional context
- Event-driven notifications when diagnostics change
