## Summary of Key Files and Classes

The rendering pipeline consists of five critical components:

1. **StyleResolver** (`third_party/blink/renderer/core/css/resolver/style_resolver.h`)
   - Resolves computed styles for DOM elements
   - Maintains a 1-1 relationship with Document
   - Uses `ResolveStyle()` to compute final styles

2. **LayoutObject** (`third_party/blink/renderer/core/layout/layout_object.h`)
   - Base class for the layout tree
   - Implements layout calculation based on computed styles
   - Inherits from DisplayItemClient to connect to painting

3. **PaintLayer** (`third_party/blink/renderer/core/paint/paint_layer.h`)
   - Represents layers in the paint tree
   - Manages special rendering properties (transforms, filters, positioning)
   - Controls painting operations for each layer

4. **PaintController** (`third_party/blink/renderer/platform/graphics/paint/paint_controller.h`)
   - Orchestrates the painting process
   - Manages display items and caching
   - Records and sequences paint operations

5. **DisplayItemList** (`third_party/blink/renderer/platform/graphics/paint/display_item_list.h`)
   - Container for display items of various types
   - Stores complete paint operation list in compact format
   - Part of PaintArtifact produced by PaintController
