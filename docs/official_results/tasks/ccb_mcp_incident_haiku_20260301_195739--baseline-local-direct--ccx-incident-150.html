## Summary

I've successfully identified the Go source files in etcd-io/raft that handle the `ErrSnapshotTemporarilyUnavailable` error during Raft log catch-up when a node attempts to rejoin a cluster.

**Key Findings:**

1. **Error Definition** (`etcd-io/raft/storage.go:38`):
   - `ErrSnapshotTemporarilyUnavailable` is defined as an error that the Storage interface returns when a snapshot is temporarily unavailable during log catch-up

2. **Snapshot Availability Check** (`etcd-io/raft/raft.go:664-676`):
   - The `maybeSendSnapshot()` function performs the snapshot availability check during Raft log catch-up
   - When a follower node falls too far behind, it calls this function to send a snapshot
   - The function checks if the snapshot returns `ErrSnapshotTemporarilyUnavailable` and gracefully handles it by returning false, allowing the Raft state machine to retry later

3. **Call Chain**:
   - `maybeSendSnapshot()` → `raftLog.snapshot()` → `Storage.Snapshot()`
   - This chain ensures that when a node is trying to catch up, the Raft library checks if the snapshot is available
