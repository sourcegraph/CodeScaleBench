### Core Components:

1. **RBACAuthorizer** - The main authorization decision engine that evaluates requests against policy rules

2. **DefaultRuleResolver** - Resolves PolicyRules by querying Role, RoleBinding, ClusterRole, and ClusterRoleBinding resources

3. **SubjectAccessEvaluator** - Determines which subjects can perform specific actions

4. **SubjectAccessReview Handlers** - REST API endpoints for:
   - `SubjectAccessReview` (cluster-wide access checks)
   - `LocalSubjectAccessReview` (namespace-scoped checks)
   - `SelfSubjectAccessReview` (current user's access)
   - `SelfSubjectRulesReview` (list user's available rules)

5. **Bootstrap Policies** - System-provided ClusterRoles and Roles defined in:
   - `policy.go` - System roles like system:admin, system:nodes
   - `namespace_policy.go` - Namespace roles like edit, view
   - `controller_policy.go` - Controller system roles

6. **Escalation Prevention** - Policy-based storage wrappers for Role, ClusterRole, RoleBinding, and ClusterRoleBinding that prevent privilege escalation by:
   - Checking `EscalationAllowed()` for system:masters members
   - Validating `ConfirmNoEscalation()` to ensure users can't grant permissions they don't have
   - Checking `BindingAuthorized()` and `RoleEscalationAuthorized()` for explicit permissions
