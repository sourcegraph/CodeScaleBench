## Objective

   - `Map` struct and its primary methods

   - Lookup, Update, Delete operations

   - Pin/Unpin operations

   - Map persistence behavior

   - MapKey, MapValue, MapPerCPUValue interfaces
   - EndpointKey as concrete example

6. **Event System**
   - Event types (MapUpdate, MapDelete)
   - DumpAndSubscribe for event streaming

7. **Collection Loading**
   - LoadCollection and LoadAndAssign functions
   - CollectionOptions

Doubles the buffer size\n  2. Retries the iteration at the same cursor position\n  3. Repeats up to `maxRetries` times (default 3)\n  4. Returns the error if max retries exceeded\n\nThis happens transparently in the iterator and doesn't restart iteration from the beginning.\n\n**Example Scenario:**\nFor an LRU hash map with uneven bucket sizes, a batch lookup might fail if one bucket is larger than the buffer. The iterator handles this by growing the buffer and retrying without losing progress.\n\n### CountAll: Count Map Entries\n\n```go\nfunc CountAll[KT, VT any, KP KeyPointer[KT], VP ValuePointer[VT]](\n    ctx context.Context, iter *BatchIterator[KT, VT, KP, VP]) (int, error)\n```\n\nHelper function that counts all entries in a map using a batch iterator.\n\n**Example:**\n\n```go\niter := NewBatchIterator[EndpointKey, EndpointValue](m)\ncount, err := CountAll(context.Background(), iter)\nif err != nil {\n    log.Fatalf(\"Count failed: %v\", err)\n}\nlog.Printf(\"Map contains %d entries\", count)\n```\n\n### BatchLookup: Low-Level Batch Lookup\n\n```go\nfunc (m *Map) BatchLookup(cursor *ebpf.MapBatchCursor, keysOut, valuesOut any,\n    opts *ebpf.BatchOptions) (int, error)\n```\n\nLow-level batch lookup operation. They survive program restarts but are lost if bpffs is unmounted.\n\n**Behavior Semantics:**\n- **Pinned maps** (created with `OpenOrCreate()` or `OpenMap()`): Persisted to bpffs, survive program restart\n- **Unpinned maps** (created with `CreateUnpinned()`): Stored in kernel memory only, released on program exit\n- **Map Recreation**: When opening an existing map, if attributes don't match, the pinned map is deleted and recreated (losing all data)\n\n### Unpin: Remove Pinned Map\n\n```go\nfunc (m *Map) Unpin() error\n```\n\nRemoves the map from the BPF filesystem. The map remains open if already open.\n\n**Behavior:**\n- Removes the pinned map file from bpffs\n- The map continues to exist in kernel memory if currently open\n- Safe to call on non-pinned maps (returns error)\n\n**Returns:** An error if the file cannot be removed.\n\n**Example:**\n\n```go\nif err := m.Unpin(); err != nil {\n    log.Fatalf(\"Unpin failed: %v\", err)\n}\n```\n\n### UnpinIfExists: Conditional Unpin\n\n```go\nfunc (m *Map) UnpinIfExists() error\n```\n\nRemoves the map from bpffs only if it exists, returning no error if it doesn't.\n\n**Returns:** An error if the unpin fails (other than the map not existing).\n\n**Example:**\n\n```go\nif err := m.UnpinIfExists(); err != nil {\n    log.Fatalf(\"UnpinIfExists failed: %v\", err)\n}\n```\n\n### Map Persistence Behavior\n\n```go\n// Scenario 1: New map\nm := NewMap(\"cilium_new\", ebpf.Hash, &Key{}, &Value{}, 1024, 0)\nm.OpenOrCreate()  // Created and pinned\nm.Close()         // Still pinned, survives restart\n\n// Scenario 2: Reopen existing map with same attributes\nm2 := NewMap(\"cilium_new\", ebpf.Hash, &Key{}, &Value{}, 1024, 0)\nm2.OpenOrCreate()  // Opens existing map, contents preserved\nm2.Close()\n\n// Scenario 3: Reopen with different attributes\nm3 := NewMap(\"cilium_new\", ebpf.Hash, &Key{}, &Value{}, 2048, 0)  // Different size!\nm3.OpenOrCreate()  // Detects incompatibility, deletes old map, creates new one\n// Previous contents are LOST\n\n// Scenario 4: Unpin and recreate\nm.Unpin()         // Remove from bpffs\nm.Recreate()      // Delete and recreate the map\n```\n\n---\n\n## Key/Value Interfaces\n\n### Implementing MapKey\n\n```go\ntype ConnectionKey struct {\n    Local  uint32\n    Remote uint32\n    Port   uint16\n}\n\nfunc (k *ConnectionKey)
