## Key Findings:

### **Core Workqueue Implementation Files:**

3. **queue.go** - Base TypedInterface with Add/Get/Done methods
4. **delaying_queue.go** - TypedDelayingInterface for delayed re-queuing

### **Key Interfaces:**
- **RateLimitingInterface**: Extends DelayingInterface with AddRateLimited, Forget, and NumRequeues methods
- **ItemExponentialFailureRateLimiter**: Per-item exponential backoff (baseDelay × 2^failures)
- **BucketRateLimiter**: Token bucket rate limiter (10 QPS, 100 bucket size)
- **Base Queue**: Add/Get/Done pattern with dirty and processing sets

### **Controller Reconciliation Loop Pattern:**
Controllers implement a standard pattern (exemplified by DeploymentController):
1. `Run()` method spawns worker goroutines
2. `worker()` calls `processNextWorkItem()` in a loop
3. `processNextWorkItem()` gets items with `queue.Get()` and marks done with `queue.Done(key)`
4. On sync failure: `queue.AddRateLimited(key)` for retry
5. On success/drop: `queue.Forget(key)` to clear rate limiter state

### **Files Located:**
- **23+ controller implementations** in `pkg/controller/` using this pattern
- **Example controller** in `staging/src/k8s.io/client-go/examples/workqueue/main.go`
- **Staging source** in `staging/src/k8s.io/client-go/util/workqueue/`
- **Duplicate** in `client-go--v0.32.0/util/workqueue/`
