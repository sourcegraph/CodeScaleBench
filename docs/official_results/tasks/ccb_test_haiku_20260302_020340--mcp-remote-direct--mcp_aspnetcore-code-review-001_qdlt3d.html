## Summary of Defects Found:

1. **CRITICAL: Unbounded Cache Growth (Memory Leak)** — Line 15 of ExpressionMemberAccessor.cs
   - The `_memberInfoCache` uses Expression objects as keys, but Expression doesn't implement value equality
   - Each new lambda expression creates a separate cache entry
   - Causes unbounded memory growth with repeated component renders
   - Fix: Remove the ineffective expression cache entirely

2. **MEDIUM: Multiple Event Subscriptions** — Line 18-23 of ExpressionMemberAccessor.cs
   - Static constructor subscribes to HotReloadManager.OnDeltaApplied without preventing multiple subscriptions
   - In hot reload scenarios, could register multiple times, causing ClearCache to be called multiple times per delta
   - Fix: Add a guard flag to ensure single subscription

3. **HIGH: Inefficient Expression Comparison** — Line 46 of DisplayName.cs
   - Uses reference equality (`For != _previousFieldAccessor`) which only works if the same Expression object is reused
   - Each new lambda in parent component creates a different object reference, defeating the optimization
   - Results in unnecessary calculations combined with cache issues
   - Fix: Remove the reference tracking and compare display name results directly

The proposed fixes in the diffs remove the problematic expression caching layer, fix the hot reload subscription logic, and simplify the component to rely on the effective MemberInfo cache and display name comparison instead.
