## Summary

**ErrSnapshotTemporarilyUnavailable** is defined in the `github.com/etcd-io/raft` module (part of the `go.etcd.io/raft/v3` dependency used by etcd). The error defines the behavior when an etcd node fails to rejoin a cluster during Raft log catch-up.

### Key Files:

1. **raft/storage.go (line 38)** - Defines `ErrSnapshotTemporarilyUnavailable` as a sentinel error returned when the Storage interface's `Snapshot()` method indicates the snapshot is not yet ready.

2. **raft/log.go** - Contains the `snapshot()` method that retrieves snapshots, checking unstable snapshots first before delegating to the storage backend.

3. **raft/raft.go** - Contains the `maybeSendAppend()` function (lines ~469-474) where the availability check occurs. When a rejoining node is too far behind, the leader attempts snapshot transfer. If `ErrSnapshotTemporarilyUnavailable` is returned, the function gracefully returns false without sending a message, allowing retry on the next Raft tick.

4. **raft/tracker/progress.go** - Manages follower progress state transitions, including `BecomeSnapshot()` which updates progress when snapshot transfer begins.

5. **raft/tracker/state.go** - Manages the `StateSnapshot` state, which pauses all replication until snapshot transfer completes.
