## Key Findings

I've identified the Go source files in grafana/grafana that implement the alerting evaluation scheduler and detect evaluation cycle overruns:

### Main Files:

1. **pkg/services/ngalert/schedule/schedule.go** - The primary scheduler implementation containing:
   - `Run()` - Entry point that starts the scheduler
   - `schedulePeriodic()` - Main event loop that processes ticks from the ticker
   - `processTick()` - Determines which rules should be evaluated on each tick
   - `runJobFn()` - **Where overrun detection occurs** (line 438)

2. **pkg/services/ngalert/schedule/ticker/ticker.go** - The periodic ticker that drives evaluations
   - `New()` - Creates the ticker
   - `run()` - Implements the ticker event loop

3. **pkg/services/ngalert/schedule/alert_rule.go** - Individual alert rule evaluation
   - `Run()` - Evaluation routine for each alert rule

4. **pkg/services/ngalert/schedule/recording_rule.go** - Recording rule evaluation (similar structure)

### Overrun Detection:

The scheduler detects when an evaluation cycle overruns its time budget in the `runJobFn()` method. When an alert rule evaluation takes longer than the scheduler's base interval, the next scheduled evaluation is dropped. This is logged as a warning:

```
"Tick dropped because alert rule evaluation is too slow"
```

This occurs at line 438 in `schedule.go` when the evaluation channel already has an unprocessed evaluation from the previous cycle.
