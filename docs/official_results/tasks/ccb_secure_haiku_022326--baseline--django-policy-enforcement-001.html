# SoftDeleteManager Usage Example

The `SoftDeleteManager` provides bulk soft-delete functionality for Django models, allowing you to mark records as deleted without removing them from the database.

## Setup

To use the `SoftDeleteManager`, add it to your model:

```python
from django.db import models

class User(models.Model):
    name = models.CharField(max_length=100)
    email = models.EmailField()
    is_deleted = models.BooleanField(default=False)
    deleted_at = models.DateTimeField(null=True, blank=True)
    deleted_by = models.CharField(max_length=100, null=True, blank=True)

    objects = models.Manager()
    soft_delete_objects = models.SoftDeleteManager()
```

## Usage

### Soft-Delete by QuerySet

```python
# Soft-delete all inactive users
inactive_users = User.objects.filter(is_active=False)
count = User.soft_delete_objects.bulk_soft_delete(
    inactive_users,
    deleted_by='admin_user'
)
print(f"Soft-deleted {count} users")
```

### Soft-Delete by IDs

```python
# Soft-delete specific users by ID
user_ids = [1, 2, 3, 4, 5]
count = User.soft_delete_objects.bulk_soft_delete(user_ids)
print(f"Soft-deleted {count} users")
```

### Restore Soft-Deleted Records

```python
# Restore all soft-deleted users
deleted_users = User.objects.filter(is_deleted=True)
count = User.soft_delete_objects.bulk_restore(deleted_users)
print(f"Restored {count} users")

# Restore specific users by ID
count = User.soft_delete_objects.bulk_restore([1, 2, 3])
print(f"Restored {count} users")
```

## Benefits

- **Audit Compliance**: Records remain in the database with timestamps, maintaining audit trails
- **Reversibility**: Soft-deleted records can be restored at any time
- **Performance**: Bulk operation using a single database UPDATE query
