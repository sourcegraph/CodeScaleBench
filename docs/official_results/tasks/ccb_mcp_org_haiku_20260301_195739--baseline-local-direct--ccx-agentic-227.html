### 2. **Stats Implementation** (source/common/stats/)

- **ThreadLocalStoreImpl/ScopeImpl**: Production stats store with thread-local optimization
- **SymbolTable & StatNamePool**: Byte-encoded stat name interning for memory efficiency
- **TagExtractorImplBase & TagProducerImpl**: Regex-based tag extraction system
- **MetricImpl, ParentHistogramImpl, ThreadLocalHistogramImpl**: Reference-counted metric storage
- **AllocatorImpl**: Symbol table interning and metric allocation

### 3. **Stat Sinks** (source/extensions/stat_sinks/)
- **UdpStatsdSink/TcpStatsdSink**: Statsd format with configurable tag formats (Datadog, Graphite)
- **GrpcMetricsServiceImpl**: gRPC-based metric export
- **OpenTelemetryImpl**: OpenTelemetry format export
- **PrometheusStatsFormatter**: Prometheus exposition format output

The architecture uses a modular design where metrics flow from creation → stat name interning → tag extraction → storage → multi-sink dispatch, enabling efficient memory usage and flexible metric distribution to multiple backends.
