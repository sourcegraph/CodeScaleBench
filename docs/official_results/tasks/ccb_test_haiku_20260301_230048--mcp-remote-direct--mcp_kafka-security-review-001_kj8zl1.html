## Summary of Findings

### 1. **Token Expiry Validation Missing** (CRITICAL)
**File**: `clients/src/main/java/org/apache/kafka/common/security/scram/internals/ScramSaslServer.java` (line 121)

**Issue**: The delegation token's expiry timestamp (`tokenExpiryTimestamp`) is retrieved from the callback but is **never validated** against the current time. This allows **expired delegation tokens to be accepted**, enabling:
- Token replay attacks
- Access by users whose credentials should have expired
- Circumvention of credential lifetime controls

**Fix**: Add an explicit expiry check immediately after retrieving the timestamp:
```java
if (tokenExpiryTimestamp != null && System.currentTimeMillis() > tokenExpiryTimestamp)
    throw new SaslException("Delegation token has expired");
```

---

### 2. **Authorization Bypass in ACL Enforcement** (CRITICAL)
**File**: `core/src/main/scala/kafka/security/authorizer/AclAuthorizer.scala` (line 377)

**Issue**: The `authorizeByResourceType` method returns `ALLOWED` based on the `shouldAllowEveryoneIfNoAclIsFound` config **before checking deny prefix ACLs**. This violates the fundamental security principle that DENY rules must always take precedence. An attacker with a deny prefix ACL could gain unauthorized access when this config is enabled.

**Fix**: Move the `shouldAllowEveryoneIfNoAclIsFound` check to **after** examining all deny ACLs (both literals and prefixes), ensuring it only applies when NO ACLs exist.
