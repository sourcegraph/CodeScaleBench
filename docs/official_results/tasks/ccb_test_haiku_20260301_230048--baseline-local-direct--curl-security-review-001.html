## Summary of Found Defects

### 1. **Critical: Heap Buffer Overflow in lib/base64.c (Line 202)**
- **Issue**: Removal of integer overflow protection in `base64_encode()`
- **Impact**: On 32-bit systems, the calculation `(insize + 2) / 3 * 4 + 1` can overflow when encoding large inputs (insize > UINT_MAX/4). This causes `malloc()` to be called with a wrapped-around small value, resulting in undersized buffer allocation and heap overflow.
- **Risk**: Remote code execution through large base64 inputs

### 2. **High: Logic Flaw in lib/cookie.c (Line 147)**
- **Issue**: Incorrect array index in cookie domain boundary validation
- **Impact**: The function `cookie_tailmatch()` is supposed to check if the character BEFORE a matched domain is a dot (subdomain boundary). The removed `-1` offset causes the code to check the first character of the matched domain instead, which can never be a dot. This breaks all legitimate subdomain cookie matching.
- **Risk**: Denial of service - prevents cookies from working correctly, breaking website functionality
