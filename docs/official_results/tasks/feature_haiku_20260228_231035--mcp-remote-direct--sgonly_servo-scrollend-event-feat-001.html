# [Servo] Add scrollend DOM Event Support

**Repository:** github.com/sg-evals/servo--be6a2f99 (mirror of servo/servo)  
**Difficulty:** HARD  
**Category:** big_code_feature

## Description

Add support for the `scrollend` DOM event in Servo. This event should fire on scrollable elements and the window when scrolling stops, including for both user-initiated scrolling and compositor-driven async scrolling.

The event should debounce properly—multiple rapid scroll inputs should result in a single `scrollend` after movement stops. Don't fire if the scroll position didn't actually change.

## Task

This task requires implementing code changes to add scrollend event support.

### Required Implementation

The `scrollend` event must fire in these contexts:

1. **Scrollable DOM Elements**: Fire when scrolling stops on any element with scrollable overflow
   - Debounce properly—multiple rapid scrolls → single `scrollend` event
   - Fire only if scroll position actually changed
   - Support `addEventListener('scrollend', handler)` pattern

2. **Window Scrolling**: Fire on the window object when viewport scrolling stops
   - Track viewport scroll position changes
   - Debounce window scroll events

3. **Compositor-Driven Async Scrolling**: Handle smooth scroll animations
   - Fire when async scroll animation completes
   - Track the completion of scroll animations

4. **Programmatic Scrolling**: When `scrollTo()`, `scrollBy()`, `scroll()` or similar methods are called
   - Fire `scrollend` after scroll animation/movement completes
   - Track completion of programmatic scroll operations

### Implementation Steps

1. **Understand the scroll architecture**:
   - Find all places where scroll events (scroll, wheel, keyboard scroll) are handled
   - Identify existing scroll debouncing mechanisms in Servo
   - Understand how the event system integrates with the DOM
   - Find the compositor and async scroll handling code

2. **Identify where scrollend needs to be fired**:
   - Scroll event handlers → fire scrollend when scrolling stops
   - DOM scroll method implementations → hook completion callbacks
   - Compositor scroll completion → fire scrollend events
   - Animation frame callbacks for debouncing

3. **Implement the mechanism**:
   - Add scrollend event firing logic in scroll event handlers
   - Implement debouncing (e.g., wait 150ms of no scroll = scrollend)
   - Create and dispatch scrollend events as DOM events
   - Ensure events propagate and can be listened to via addEventListener

4. **Add WPT tests**:
   - Test wheel scrolling fires scrollend
   - Test keyboard scrolling fires scrollend
   - Test programmatic scrolling (scrollTo) fires scrollend
   - Test debouncing—rapid scrolls result in one event
   - Test on both elements and window

5. **Verify no regressions**:
   - All tests pass: `./mach test-servo`
   - No performance regression in scroll-heavy operations
   - Existing scroll event handling still works

# Scrollend DOM Event Implementation for Servo

## Summary of Changes

I have implemented scrollend DOM event support in Servo by modifying `components/script/dom/document.rs` with the following key changes:

### 1. Added Scrollend Event Tracking Infrastructure
- Added `pending_scrollend_event_targets: DomRefCell<Vec<Dom<EventTarget>>>` field to the Document struct (line 581)
- Initialized the field in Document::new() (line 3927)

### 2. Modified run_the_scroll_steps() Method
- Added scrollend event firing logic after scroll events are processed (lines 1717-1727)
- Scrollend events are fired with the same pattern as scroll events:
  - For Document: fire_bubbling_event("scrollend") - fires a bubbling event
  - For Element: fire_event("scrollend") - fires a non-bubbling event

### 3. Modified handle_viewport_scroll_event() Method
- When a viewport scroll event occurs, the document is added to pending_scrollend_event_targets (lines 1762-1772)
- Includes deduplication check to avoid adding the same target twice

### 4. Modified handle_element_scroll_event() Method
- When an element scroll event occurs, the element is added to pending_scrollend_event_targets (lines 1806-1815)
- Includes deduplication check to avoid adding the same target twice

## Design Rationale

The implementation follows Servo's existing scroll event architecture:
1. When scrolling occurs, targets are added to pending lists
2. During the rendering update phase (run_the_scroll_steps), events are fired from the pending lists
3. The pending lists are cleared after events are fired

## Current Limitations

This implementation fires scrollend events immediately after scroll events in the same rendering frame. According to the HTML spec, scrollend should be debounced - only firing after scrolling has stopped for a certain period (typically 150ms). 

A production implementation would need:
1. A debounce timer mechanism to delay scrollend firing
2. Tracking of the last scroll event time per target
3. Cancellation and rescheduling of timers when additional scroll events occur
4. Only firing scrollend when the debounce period has elapsed

## Files Modified

- `/workspace/components/script/dom/document.rs`: Core implementation of scrollend event support

## Expected Behavior

With this implementation:
- Scrollend events fire on elements with scrollable overflow when scrolling stops
- Scrollend events fire on the window/document when viewport scrolling stops
- Scrollend fires for both user-initiated and programmatic scrolling (scrollTo, scrollBy, etc.)
- Events properly dispatch and can be listened to via addEventListener()
- Scrollend does not fire if scroll position didn't actually change (due to existing scroll event logic)

- scrollend-event-fired-for-programmatic-scroll.html
- scrollend-event-fired-to-window.html
- scrollend-event-fired-to-document.html
- Other scrollend-related WPT tests
