## Summary

The `ErrSnapshotTemporarilyUnavailable` error is part of the Raft snapshot availability check mechanism that occurs when an etcd node fails to rejoin a cluster because its local log is too old.

**Key Findings:**

1. **Error Definition**: `ErrSnapshotTemporarilyUnavailable` is defined in the raft library (`go.etcd.io/raft/v3/storage.go`), which etcd depends on.

2. **Critical File**: `server/etcdserver/bootstrap.go` contains the `MemoryStorage()` method (lines 700-712) that creates the raft.MemoryStorage instance. This is the etcd-io/etcd wrapper that provides snapshot availability to the raft library.

3. **Snapshot Availability Check**: During Raft log catch-up when a slow follower (rejoining node) is too far behind:
   - The raft library's `maybeSendSnapshot()` function calls `Storage.Snapshot()`
   - If the snapshot is temporarily unavailable (still being prepared), it returns `ErrSnapshotTemporarilyUnavailable`
   - Instead of panicking, the leader gracefully retries on the next heartbeat tick

4. **Related Components**:
   - `raftNode` struct in `server/etcdserver/raft.go` uses the MemoryStorage
   - Bootstrap chain: `bootstrapWALFromSnapshot` → `bootstrappedWAL.MemoryStorage()` → `newRaftNode`
