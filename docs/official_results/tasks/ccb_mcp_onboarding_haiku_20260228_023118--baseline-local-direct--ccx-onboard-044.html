## Key Findings

**1. InferCtxt struct** - `compiler/rustc_infer/src/infer/mod.rs`
- The main inference context that coordinates all type inference operations
- Contains type variable storage, region constraints, unification tables, and obligation tracking
- Serves as the central hub for type checking throughout the compiler

**2. FnCtxt struct** - `compiler/rustc_hir_typeck/src/fn_ctxt/mod.rs`
- The function-level type checker that wraps InferCtxt
- Manages function-specific state like body_id, parameter environment, return type coercion, and divergence tracking
- Responsible for type checking function bodies, closures, and const expressions

**3. TypeVariableOrigin struct** - `compiler/rustc_infer/src/infer/type_variable.rs`
- Tracks the origin of type variables created during inference
- Contains a span and optional param_def_id for diagnostic purposes
- Used to track which type parameter a variable was instantiated for

These components work together in a hierarchical manner: **InferCtxt** provides the low-level inference machinery, **TypeVariableOrigin** tracks metadata about inference variables, and **FnCtxt** wraps everything for function-level type checking with additional context management.
