## Summary of Analysis

I traced the complete pricing flow from `BarrierOption.NPV()` through:

1. **Lazy Evaluation Chain**: NPV() → calculate() → LazyObject.calculate() → performCalculations() → engine.calculate()

2. **Two Pricing Strategies**:
   - **Analytic**: AnalyticBarrierEngine directly queries term structures and applies Haug's closed-form formulae
   - **Monte Carlo**: MCBarrierEngine inherits from McSimulation, generates sample paths, and prices them

3. **Path Generation Pipeline**:
   - RNG (PseudoRandom/LowDiscrepancy) → Gaussian sequence → Brownian Bridge transform → PathGenerator
   - PathGenerator iteratively calls GeneralizedBlackScholesProcess.evolve()

4. **Process Evolution**:
   - GeneralizedBlackScholesProcess queries YieldTermStructure (risk-free, dividend) and BlackVolTermStructure
   - Implements SDE: d(ln S) = (r-q-σ²/2)dt + σ dW_t

5. **Key Design Patterns**:
   - Lazy Object (caching + observer)
   - Strategy (multiple engine implementations)
   - Template Method (McSimulation algorithm structure)
   - Traits (RNG policy selection)
   - Handle/Body (term structure references)

The document includes detailed dependency chains, component responsibilities table, interface contracts, and explains how all 450K lines are orchestrated to price a single barrier option.
