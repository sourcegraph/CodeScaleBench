## Dashboard JSON Model Versioning and Migration in Grafana

The Grafana dashboard JSON model versioning system is implemented in **`apps/dashboard/pkg/migration/`** with the following key components:

### Core Migration Framework:
- **`migrate.go`**: Main entry point with `Migrate()` function that orchestrates schema version migrations
- **`frontend_defaults.go`**: Functions that replicate frontend behavior (`applyFrontendDefaults()`, `applyPanelDefaults()`, `cleanupDashboardForSave()`, etc.)

- **`migrations.go`**: Defines `SchemaVersionMigrationFunc` type and `GetMigrations()` function that maps version numbers (2-42) to migration functions. Latest version is 42.
- **`v2.go` through `v42.go`**: Individual field-level migration functions (`V2`, `V3`, ..., `V42`) that handle each schema version upgrade

### Migration Utilities:
- **`migration_utils.go`**: Helper functions for safe type conversions (`GetStringValue()`, `GetBoolValue()`, `ConvertToFloat()`, etc.)
- **`datasource_utils.go`**: Datasource reference migration utilities (`GetDataSourceRef()`, `MigrateDatasourceNameToRef()`)
