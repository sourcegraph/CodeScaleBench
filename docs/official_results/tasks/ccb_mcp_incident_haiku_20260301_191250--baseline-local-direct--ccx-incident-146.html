### **Files Found:**
1. **django/db/models/query.py** - Contains the QuerySet class with lazy evaluation mechanisms
2. **django/db/models/sql/query.py** - Contains the Query class that lazily stores SQL parameters
3. **django/db/models/sql/compiler.py** - Contains SQLCompiler that compiles and executes SQL

### **Lazy SQL Compilation & Execution Chain:**

When a QuerySet is first iterated, the following chain executes:

1. **QuerySet.__iter__()** (line 2299) - Initiates iteration
2. **QuerySet._fetch_all()** (line 2285) - Checks if results are cached
3. **QuerySet.iterator()** (line 2313) - Returns ModelIterable
4. **ModelIterable.__iter__()** (line 91) - Gets the SQL compiler and executes the query
5. **Query.get_compiler()** (line 364) - Creates a SQLCompiler instance
6. **SQLCompiler.execute_sql()** (line 1601) - Triggers actual database query
7. **SQLCompiler.as_sql()** (line 763) - Compiles the SQL string
8. **cursor.execute()** (line 1631) - Executes SQL and **raises OperationalError if database connection fails**

The key insight is that Django stores query metadata in the Query object without compiling it to SQL. This deferred compilation only happens when the QuerySet is first evaluated (iterated), allowing for efficient query optimization and chaining.
