# Envoy HTTP Connection Manager: Architecture Document

## Executive Summary

The **HTTP Connection Manager (HCM)** is the core subsystem in Envoy responsible for processing HTTP requests from their arrival on a downstream connection through to upstream dispatch, and back again. It orchestrates four major components:

1. **ConnectionManagerImpl** — Owns the connection lifecycle and manages codec instantiation
2. **FilterManager** — Executes the HTTP filter chain for request and response processing
3. **Router Filter** — Selects upstream clusters and dispatches requests
4. **ClusterManager & Upstream** — Manages connection pools and load balances across cluster hosts

This document explains the **design**, **data flow**, **extension points**, and **error handling** of the HCM, focusing on how these components interact to process an HTTP request end-to-end.
