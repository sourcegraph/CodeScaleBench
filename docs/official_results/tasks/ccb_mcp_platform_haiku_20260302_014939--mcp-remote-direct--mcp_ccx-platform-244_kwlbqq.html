## Summary

The answer includes **14 C++ source files** that implement the hot restart mechanism across the following components:

**Core Architecture:**
- **HotRestart Interface** - Defines the abstraction for live process restart
- **HotRestartImpl** - Main Linux implementation managing shared memory, process coordination
- **HotRestartingParent** - Handles parent process socket/stats/drain requests
- **HotRestartingChild** - Handles child process initialization and parent communication
- **HotRestartingBase + RpcStream** - Low-level Unix domain socket communication

**Key Mechanisms Covered:**
1. **Parent/Child Handshake** - Via Unix domain sockets (AF_UNIX, SOCK_DGRAM)
2. **Socket Passing** - Using SCM_RIGHTS ancillary data in sendmsg()
3. **Drain Sequence** - Parent gracefully closes listeners and forwards unmatched QUIC packets to child
4. **Child Readiness** - Signaled through drainParentListeners() and shared memory initialization flags
5. **Shared Memory** - Process-shared pthread mutexes and version checking
6. **Stats Merging** - Parent exports gauges and counter deltas to child
7. **UDP Forwarding** - Separate RPC stream for QUIC packet routing during drain

The implementation uses Protocol Buffers for message serialization and supports up to 3 concurrent Envoy processes for hot restart.
