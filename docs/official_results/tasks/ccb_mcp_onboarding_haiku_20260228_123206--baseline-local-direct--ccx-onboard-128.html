## HotSpot JIT Compilation Pipeline Components

All files are located in `src/hotspot/share/`:

1. **CompileBroker** (`compiler/compileBroker.hpp`) - The broker that manages all compilation requests and coordinates between the application threads and compilation threads.

2. **CompilerThread** (`compiler/compilerThread.hpp`) - The thread type that runs the actual compilation work.

3. **AbstractCompiler** (`compiler/abstractCompiler.hpp`) - The base class that defines the common interface for both C1 and C2 compilers.

4. **C1 Compiler** (`c1/c1_Compiler.hpp`) - The `Compiler` class implementing the first-tier (tier 1) JIT compiler for fast code generation with minimal optimization.

5. **C2 Compiler** (`opto/c2compiler.hpp`) - The `C2Compiler` class implementing the second-tier (tier 2) optimizing JIT compiler with advanced optimizations.

The compilation pipeline works as follows:
- CompileBroker receives compilation requests from the application
- It assigns these requests to CompilerThread instances
- These threads dispatch work to either C1 or C2 compiler implementations through the AbstractCompiler interface
- C1 provides quick compilation for frequently executed code
- C2 performs deeper analysis and optimization on hot code paths
