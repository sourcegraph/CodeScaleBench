## Key Findings:

### Internal Layer (lib/internal/child_process.js):
- **ChildProcess class** (line 254): Manages the child process lifecycle
- **ChildProcess.prototype.spawn()** (line 354): Bridges JavaScript to C++ via the Process binding

### C++ Native Bindings:
- **process_wrap.cc**: ProcessWrap class with Spawn() method that handles the async spawn operation
- **spawn_sync.cc**: SyncProcessRunner with ParseOptions(), CopyJsString(), and CopyJsStringArray() methods for synchronous spawning
- **Permission validation**: Built-in checks before process spawning is allowed

### Security Mechanism:
The primary command injection defense is **null-byte validation** at the JavaScript/C++ boundary. By checking for and rejecting null bytes in:
- Executable file paths
- Command arguments
- Environment variable names and values

This prevents attackers from truncating or injecting commands through null-byte injection attacks.
