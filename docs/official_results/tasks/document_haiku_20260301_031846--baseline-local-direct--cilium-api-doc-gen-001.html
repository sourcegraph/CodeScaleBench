## Core Concepts

### Design Philosophy

Cilium's eBPF map API provides type-safe Go wrappers around kernel eBPF maps. The API emphasizes:

- **Type Safety**: Keys and values must implement `MapKey` and `MapValue` interfaces
- **Lifecycle Management**: Maps have explicit creation, opening, and closing semantics
- **Persistence**: Maps can be pinned to the BPF filesystem for reuse across process restarts
- **Synchronization**: Optional cache tracking ensures consistency between userspace and kernel
- **Event Streaming**: Built-in event buffer for auditing map operations

### Supported Map Types

The API supports the following eBPF map types via `ebpf.MapType`:

- `Hash`: Standard hash map
- `Array`: Fixed-size array
- `PerCPUHash`: Per-CPU hash map
- `PerCPUArray`: Per-CPU array
- `LRUHash`: LRU eviction hash map
- `LRUCPUHash`: LRU per-CPU hash map
- `LPMTrie`: Longest prefix match trie
- `RingBuffer`: Ring buffer for events
- `ProgramArray`: Program arrays for tail calls
- `PerCPUCGroupStorage`: Per-CPU cgroup storage
