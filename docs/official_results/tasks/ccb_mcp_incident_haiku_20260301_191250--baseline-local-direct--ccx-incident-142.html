I've successfully analyzed the Kafka repository and found the key Java source files related to consumer heartbeat session timeouts. Here's a summary of my findings:

## Key Findings:

**Files Identified:**
1. **`Heartbeat.java`** - Defines the `sessionTimeoutExpired()` method that checks if the heartbeat session has timed out
2. **`AbstractCoordinator.java`** - Contains the `HeartbeatThread` class that runs the heartbeat loop and checks for session timeout expiration during consumer polling
3. **`GroupRebalanceConfig.java`** - Stores the `sessionTimeoutMs` configuration value
4. **`ConsumerConfig.java`** - Defines the session timeout configuration with a default value of 45 seconds (45000 ms)

**Timeout Threshold Definition:**
- The session timeout threshold is configured in `ConsumerConfig.java` with the configuration key `session.timeout.ms` and a default value of 45000 milliseconds
- This value is passed through `GroupRebalanceConfig` to the `Heartbeat` class

**Error Detection and Logging:**
- In `AbstractCoordinator.java` (lines 1513-1517), within the `HeartbeatThread.run()` method's consumer polling loop, the timeout is checked via `heartbeat.sessionTimeoutExpired(now)`
- When the session timeout expires, it calls `markCoordinatorUnknown("session timed out without receiving a heartbeat response")` which logs an info message and disconnects from the coordinator, causing a consumer group rebalance
