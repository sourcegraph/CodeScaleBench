## Summary

I've completed a comprehensive architectural analysis of the QuantLib barrier option pricing chain and written it to `/logs/agent/solution.md`. Here are the key findings:

### Main Discovery: Two Parallel Paths

1. **Analytic Path**: Direct closed-form formulas (Haug's option pricing formulas)
   - Queries term structures once at maturity
   - Combines Black-Scholes cumulative normal with barrier adjustments

2. **Monte Carlo Path**: Multi-layer framework
   - McSimulation orchestrates convergence (tolerance/samples)
   - MonteCarloModel loops: generate paths → price → accumulate statistics
   - PathGenerator evolves the stochastic process using random sequences
   - GeneralizedBlackScholesProcess queries term structures at each time step
   - BarrierPathPricer checks barrier conditions and applies payoffs

### Architecture Patterns

- **Lazy Evaluation**: LazyObject caches results, invalidates on market data changes
- **Strategy Pattern**: Engines are swappable (analytic, MC, FD, binomial)
- **Policy-Based Templates**: MC engines specialize on RNG and statistics policies
- **Handle/Body**: Term structures deferred via smart pointers
