## Summary

The Kafka consumer group coordinator (`GroupMetadataManager`) is responsible for managing group membership, member state transitions, partition assignment, and offset commits. While `GroupMetadataManagerTest.java` provides substantial coverage of the happy path and basic error cases, several critical failure modes related to member epoch validation, group size constraints, state consistency, and concurrent operations lack adequate test coverage. This analysis identifies 7 specific gaps that could lead to runtime failures in production.

## Failure Modes and Proposed Tests
