## Investigation Summary

**Root Cause Identified:** The V38 backend migration in Grafana contains conditional logic at `apps/dashboard/pkg/migration/schemaversion/v38.go` (line 117) that only processes `fieldConfig.defaults.custom` when it already exists in the saved dashboard JSON.

**The Asymmetry:**
- **Defaults processing:** Conditional on `defaults.custom` existing (line 115-127)
- **Overrides processing:** Always executes, regardless of whether `defaults.custom` exists (line 131)

This creates a critical issue when combined with the cleanup phase in `frontend_defaults.go`:

1. Dashboards without explicit `fieldConfig.defaults.custom` are not marked with the `_originallyHadFieldConfigCustom` flag (line 1155-1156)
2. During cleanup, this flag determines whether fieldConfig is preserved
3. If fieldConfig is removed as a "default value," field overrides are lost

**Evidence:** The comprehensive test case at `v38_test.go` (lines 443-599) explicitly tests this scenario and shows the expected behavior—that overrides should be migrated even without explicit `defaults.custom`.

**Why Explicit `defaults.custom` Works:** Dashboards that explicitly save `fieldConfig.defaults.custom` set the preservation flag during migration tracking, ensuring the fieldConfig structure (and all overrides) survive the cleanup phase.

## Summary

The table panel field override configuration regression in v10.4 stems from the conditional logic in the V38 backend migration that only processes `fieldConfig.defaults.custom` when it explicitly exists in the saved dashboard JSON. When dashboards lack explicit `fieldConfig.defaults.custom`, the migration skips creating the defaults structure, while simultaneously migrating field overrides that reference `custom.displayMode`. if defaults, ok := fieldConfig["defaults"].(map[string]interface{}); ok {

```go
        // ONLY processes if custom exists
        if displayMode, exists := custom["displayMode"]; exists {
            custom["cellOptions"] = migrateTableDisplayModeToCellOptions(displayModeStr)
            delete(custom, "displayMode")
        }
    }
}
```

