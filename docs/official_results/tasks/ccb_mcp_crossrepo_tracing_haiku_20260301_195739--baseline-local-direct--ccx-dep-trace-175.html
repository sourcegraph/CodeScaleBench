## Pipeline Overview

The alertmanager notification pipeline consists of these key components:

### 1. **Alert Ingestion** (`api/v2/api.go`)
- The `postAlertsHandler` function (line 328) receives and validates incoming alerts
- Converts OpenAPI format to internal alert types and stores them in the alert provider

### 2. **Routing Tree Traversal** (`dispatch/dispatch.go` & `dispatch/route.go`)
- The `Dispatcher.routeAlert` function (line 274) processes each alert
- Calls `Route.Match` (line 160 in route.go) which implements a depth-first left-to-right search through the routing tree
- Determines which receivers should be notified based on label matching

### 3. **Alert Aggregation** (`dispatch/dispatch.go`)
- The `groupAlert` function (line 443) batches alerts into aggregation groups
- Groups are created based on the route's GroupBy configuration
- Aggregation groups flush at configured intervals (GroupWait, GroupInterval)

### 4. **Notification Pipeline** (`notify/notify.go` & `notify/mute.go`)
- `PipelineBuilder.New` (line 434) constructs the notification pipeline
- Pipeline executes sequentially through multiple stages:
  - GossipSettleStage (cluster sync)
  - **Inhibition Check** - `Inhibitor.Mutes` (line 183 in `inhibit/inhibit.go`)
  - Time Active/Mute interval checks
  - **Silence Check** - `Silencer.Mutes` (line 163 in `silence/silence.go`)
  - Receiver dispatch stages

### 5. **Suppression Filtering** (`notify/mute.go`)
- The `MuteStage.Exec` function (line 58) filters suppressed alerts
- Uses the Muter interface to check inhibition and silence rules
- Removes muted alerts before final notification dispatch
