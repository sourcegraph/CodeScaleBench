**39 C++ source files** implement HTTP connection manager route matching, organized into these categories:

1. **HTTP Connection Manager** - The network filter entry point
2. **Connection Manager Implementation** - Core ConnectionManagerImpl managing HTTP streams

4. **Router Filter** - The Filter class that executes matched routes and dispatches to upstreams

### Key Classes:
- **RouteMatcher** - Virtual host selection (`findVirtualHost()`) and route matching (`route()`)
- **VirtualHostImpl** - Route matching within a virtual host
- **ConfigImpl** - Main route configuration interface
- **RouteEntryImplBase** - Base for route matching implementations (Prefix, Path, Regex, etc.)
- **ConnectionManagerImpl** - HTTP connection management
- **Filter** (router) - Route action dispatch to upstreams

### Route Matching Flow:
ConnectionManagerImpl → RouteMatcher::findVirtualHost() → VirtualHostImpl → RouteEntry matching → Router::Filter → UpstreamRequest

The files also include supporting infrastructure for:
- Dynamic route discovery (RDS/VHDS)
- Header manipulation and parsing
- Metadata-based routing

- Rate limiting and retry logic
