# Istio Control Plane Developer Onboarding Guide

Welcome to Istio's control plane development! This guide will help you understand how to build Istio, navigate the codebase, and make your first contribution.

## Part 1: Build Prerequisites and Setup

### Required Go Version

Istio requires **Go 1.25.0** or later. Verify your Go installation:

```bash
go version
```

If you need to install or update Go, visit [golang.org/dl](https://golang.org/dl).

### Clone the Repository with Submodules

Clone the Istio repository and initialize all submodules:

```bash
git clone https://github.com/istio/istio.git
cd istio
git submodule update --init --recursive
```

### Build System Overview

Istio uses a Makefile-based build system. The main entry point is `/workspace/Makefile`, which delegates to `Makefile.core.mk` for the actual build logic. The build system can run natively on your system or inside a Docker container.

**Option 1: Native Build (Recommended for Development)**

No special setup needed. The build system will automatically configure your environment via `common/scripts/setup_env.sh`.

**Option 2: Container-Based Build**

To build entirely in a Docker container:

```bash
make BUILD_WITH_CONTAINER=1 build
```

This approach only requires `make` and Docker, with no other dependencies on your host.

### Building Core Control Plane Binaries

#### Build All Binaries

```bash
make build
```

This builds all Go binaries including:
- `pilot-discovery`: The Istio control plane service
- `pilot-agent`: Sidecar proxy agent for workload pods
- `istioctl`: Command-line tool for managing Istio
- Various test and sample binaries

Binaries are output to `out/$(uname -s)_$(uname -m)/` (e.g., `out/linux_amd64/` on Linux x86_64).

#### Build for Linux Only

```bash
make build-linux
```
