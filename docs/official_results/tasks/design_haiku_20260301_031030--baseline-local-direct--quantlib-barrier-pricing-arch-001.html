   - `YieldTermStructure` provides discount factors for r(t) and q(t)
   - `BlackVolTermStructure` provides σ(t,K) for path diffusion
   - Observable/Observer pattern enables automatic recalculation on market data changes

   - Lazy evaluation with caching
   - Strategy pattern for alternative engines
   - Template method for MC framework

Analysis\n\n### Design Patterns Identified\n\n1. **Lazy Evaluation with Caching**: The `LazyObject` pattern enables on-demand calculation where results are cached until dependencies change. The `update()` method notifies dependents when input structures (term structures, processes) change.\n\n2. **Strategy Pattern**: `PricingEngine` is an abstract interface with multiple implementations (AnalyticBarrierEngine, MCBarrierEngine). The instrument delegates all pricing logic to the engine without knowing implementation details.\n\n3. **Template Method Pattern**: `McSimulation` defines the MC algorithm's skeleton (calculate, value, valueWithSamples) while subclasses implement specific methods (pathGenerator, pathPricer, timeGrid).\n\n4. **Trait Classes**: `MCTraits` (SingleVariate, MultiVariate) and RNG policies are used to customize MC behavior at compile-time without runtime overhead.\n\n5. **Handle Pattern**: Uses smart pointers (`Handle<>`) to term structures and processes, enabling observable linking and automatic recalculation when market data changes.\n\n### Component Responsibilities\n\n**BarrierOption (Instrument)**\n- Stores barrier parameters (type, level, rebate)\n- Implements `setupArguments()` to populate engine arguments\n- Delegates all valuation to the pricing engine via `NPV()` call\n\n**PricingEngine Hierarchy**\n- `AnalyticBarrierEngine`: Computes closed-form barrier option prices using Haug formulas\n  - Direct queries of term structures for drift, discount, volatility\n  - No path simulation required\n  - Fast but limited to specific models (Black-Scholes)\n\n- `MCBarrierEngine`: Monte Carlo simulation for barrier options\n  - Creates PathGenerator and PathPricer on demand\n  - Manages simulation parameters (steps, samples, tolerance, brownian bridge, antithetic variates)\n  - Flexible: can price with any stochastic process\n\n**GeneralizedBlackScholesProcess**\n- Encapsulates stochastic evolution dynamics: `dx = (r-q-\u03c3\u00b2/2)dt + \u03c3 dW`\n- Acts as **data provider** for term structures:\n  - Queries `YieldTermStructure` for risk-free rate r(t) and dividend yield q(t)\n  - Queries `BlackVolTermStructure` for volatility \u03c3(t, K)\n- Implements `evolve()` to step the state forward: `x[i+1] = f(x[i], random_shock)`\n- Observable: notifies engines when underlying, rates, or volatility change\n\n**PathGenerator**\n- Converts random number sequence to asset price path\n- Uses `StochasticProcess.evolve()` to compute increments\n- Supports variance reduction: Brownian Bridge, antithetic variates\n- Works with any 1-D process (not tied to Black-Scholes)\n\n**PathPricer**\n- Computes option value for a single simulated path\n- BarrierPathPricer: checks barrier levels, applies payoff, discounts cash flows\n- Decoupled from path generation (single responsibility)\n\n**Term Structures**\n- `YieldTermStructure`: provides discount factors D(t) = e^{-\u222br(s)ds}\n  - Used for: risk-free discounting, dividend yield discounting\n  - Observable: when evaluationDate() or market data changes, notifies dependents\n\n- `BlackVolTermStructure`: provides implied volatility \u03c3(t, K)\n  -
