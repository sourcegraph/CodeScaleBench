The Cilium eBPF map API provides comprehensive Go wrappers for user-space interaction with kernel eBPF maps. This document serves as a complete reference for developers working with eBPF maps in Cilium.

The API is organized around the `Map` type, which wraps the underlying kernel map and provides high-level operations like CRUD, iteration, pinning, event subscription, and error recovery.

## Core Map Types and Interfaces

### MapKey Interface

```go
type MapKey interface {
    fmt.Stringer
    // New returns a pointer to a new instance of this MapKey.
    // Used to allocate memory for iteration and lookups.
    New() MapKey
}
```

**Purpose**: Defines the key type for a BPF map. All keys must implement `fmt.Stringer` for logging and string representation.

**Example Implementation**:
```go
type MyKey struct {
    ID uint32
}

func (k *MyKey) New() MapKey {
    return &MyKey{}
}

func (k *MyKey) String() string {
    return fmt.Sprintf("MyKey{ID: %d}", k.ID)
}
```
