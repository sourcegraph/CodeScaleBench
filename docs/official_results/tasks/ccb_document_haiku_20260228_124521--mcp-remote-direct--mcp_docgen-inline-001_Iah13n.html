Full-page cache middleware for Django.

This module provides a two-part caching system for caching entire Django-generated
pages and serving them without reprocessing. The middleware works by storing the
rendered HTML response in Django's cache backend and retrieving it on subsequent
requests for the same URL.

Middleware Architecture
=======================

The canonical way to enable cache middleware is to set ``UpdateCacheMiddleware``
as your first piece of middleware, and ``FetchFromCacheMiddleware`` as the last::

    MIDDLEWARE = [
        'django.middleware.cache.UpdateCacheMiddleware',
        ...,
        'django.middleware.cache.FetchFromCacheMiddleware'
    ]

This ordering is counterintuitive but correct:

* ``UpdateCacheMiddleware`` needs to run last during the response phase, which
  processes middleware bottom-up. This ensures it wraps around all other
  middleware and can cache the final response.
* ``FetchFromCacheMiddleware`` needs to run last during the request phase, which
  processes middleware top-down. This ensures it runs after all other middleware
  has processed the request, making the cache lookup the final step.

cache key does not need to be affected by other middleware. However, if any other
middleware needs to affect the cache key (such as ``LocaleMiddleware``), you must
use the two-part ``UpdateCacheMiddleware`` and ``FetchFromCacheMiddleware``.
