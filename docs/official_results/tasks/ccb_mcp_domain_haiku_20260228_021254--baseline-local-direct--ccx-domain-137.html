## Android View Rendering Pipeline - Key Files Found

**1. View.java** (`core/java/android/view/View.java`)
   - Core methods: `measure()`, `layout()`, `draw()`
   - These are called recursively on the view hierarchy

**2. ViewRootImpl.java** (`core/java/android/view/ViewRootImpl.java`)
   - Orchestrates the entire traversal: `performTraversals()`, `performMeasure()`, `performLayout()`, `performDraw()`
   - Bridges WindowManager and the View hierarchy

**3. Choreographer.java** (`core/java/android/view/Choreographer.java`)
   - Drives frame scheduling with VSYNC: `scheduleFrameLocked()`, `doFrame()`
   - Synchronizes rendering to display refresh rate

**4. ThreadedRenderer.java** (`core/java/android/view/ThreadedRenderer.java`)
   - Extends HardwareRenderer
   - Manages the rendering thread for GPU acceleration

**5. HardwareRenderer.java** (`graphics/java/android/graphics/HardwareRenderer.java`)
   - GPU-accelerated rendering bridge
   - Contains `drawRenderNode()` to replay display lists on GPU

**6. RecordingCanvas.java** (`graphics/java/android/graphics/RecordingCanvas.java`)
   - Records drawing operations for deferred GPU rendering
   - Used during View.draw() phase

## Rendering Flow:
```
Choreographer.doFrame() 
  → ViewRootImpl.performTraversals()
    → performMeasure() → View.measure()
    → performLayout() → View.layout()
    → performDraw() → View.draw(RecordingCanvas)
      → ThreadedRenderer.draw()
        → HardwareRenderer.drawRenderNode()
          → GPU rendering and Surface submission
```
