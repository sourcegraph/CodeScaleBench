   - `get_filters()` method (line 175) - **Critical control point** where `has_output()` is checked at line 221

   - `admin_list_filter()` tag (lines 521-529) - Renders individual filter specifications

4. **`django/contrib/admin/templates/admin/change_list.html`** - Template
   - Line 72: `{% for spec in cl.filter_specs %}{% admin_list_filter cl spec %}{% endfor %}`

### **Key Finding: Empty Filter Prevention**

The critical control point is **line 221 of `main.py`**:
```python
if spec and spec.has_output():
    filter_specs.append(spec)
```

Empty related-field filters are filtered out here. `RelatedFieldListFilter.has_output()` returns `False` when there are no related objects (or only an empty choice), preventing the filter from being added to `filter_specs` and therefore never reaching the template.
