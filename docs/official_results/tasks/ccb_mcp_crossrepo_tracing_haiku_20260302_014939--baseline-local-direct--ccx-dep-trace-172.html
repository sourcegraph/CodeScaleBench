## Key Findings

**Controller Registration Pattern in Kubernetes:**

The kubernetes repository implements controller registration through a multi-layered system:

1. **ControllerDescriptor Registration** - Controllers are registered in `cmd/kube-controller-manager/app/controllermanager.go` via `ControllerDescriptor` structs that contain an `InitFunc` (the initialization function for each controller).

2. **Controller Interface** - Defined in `staging/src/k8s.io/controller-manager/controller/interfaces.go`, this is the base interface that all controllers must implement, requiring a `Name()` method.

3. **StartController Orchestration** - The `StartController()` function in the manager coordinates:
   - Validating feature gates
   - Calling the controller's InitFunc to create and launch the controller
   - Registering optional debugging and health check handlers

4. **Reconciliation Loop Pattern** - Controllers like the Deployment Controller implement:
   - Informer event handlers to watch resources and queue changes
   - Worker goroutines that process queued items
   - A `syncHandler` (e.g., `syncDeployment`) that performs the actual reconciliation

5. **Complete Registration Chain**: `Run()` → `StartControllers()` → `StartController()` → `InitFunc()` → `NewController()` → `Run()` → `worker()` → `syncHandler()`
