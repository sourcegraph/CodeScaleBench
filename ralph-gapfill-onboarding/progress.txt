## Codebase Patterns
- Onboarding tasks: benchmarks/ccb_onboarding/{task_id}/ — new suite
- Three task subtypes: codebase_orientation, team_handoff, workflow_discovery
- Output format: /logs/agent/onboarding.md with numbered answers to questions (orientation) or structured handoff doc (handoff)
- Scoring: weighted checklist via ground_truth.json — per-question scoring for orientation, per-section scoring for handoff
- Orientation questions: main entry point, core packages, config loading, test structure, how to extend
- Handoff sections: purpose, dependencies (upstream/downstream), key files, failure modes, testing, debugging
- Workflow discovery: build commands, test commands, CI pipeline, PR process
- instruction.md: pose questions WITHOUT hints about file locations (orientation), or present handoff scenario (handoff)
- ground_truth.json: per-question/section findings with regex patterns matching key concepts and file paths
- ground_truth.json structure: weights object + arrays (required_findings, file_references, causal_chain, negative_checks), each item has patterns (array), weight (float), description, section
- Dockerfile: clone target repo at pinned commit, WORKDIR /workspace
- Config: configs/onboarding_2config.sh (created by infra Ralph)
- MCP advantage: list_files for structure discovery, nls_search for finding entry points, read_file for understanding patterns, find_references for downstream discovery
- test.sh verifier: fully reusable across orientation and handoff tasks — same Python scorer logic works for both
