## Codebase Patterns

### SWE-bench Pro Task Structure (SOURCE tasks — read from these)
- Tasks in `benchmarks/ccb_swebenchpro/tasks/instance_{org}-{repo}-{commit_hash}[-v{variant}]/`
- `tests/config.json` contains:
  - `patch`: the reference fix as a git diff (THIS is what becomes reference_fix.patch)
  - `test_patch`: additional test setup patches
  - `base_commit`: the pre-fix (buggy) commit
  - `fail_to_pass`: test names that should pass after fix
  - `pass_to_pass`: tests that must remain passing
  - `FAIL_TO_PASS` / `PASS_TO_PASS`: alternate key names (check both)
- `instruction.md`: problem statement (may contain file path hints — MUST be rewritten for navprove)
- `environment/Dockerfile`: uses pre-built base images `jefzda/sweap-images:{repo}-{instance_id}`
- `task.toml`: has `repo`, `language`, `difficulty`, `pre_fix_rev`, `before_repo_set_cmd`

### Navprove Task Structure (TARGET tasks — write to these)
- Tasks in `benchmarks/ccb_navprove/navprove-{short_name}-NNN/`
- Each task needs:
  1. `tests/reference_fix.patch` — extracted from source's `tests/config.json` `patch` field
  2. `environment/Dockerfile` — can reuse source's base image, add test framework if needed
  3. `instruction.md` — SYMPTOM-ONLY rewrite (see leakage rules below)
  4. `task.toml` — already scaffolded, may need `source_instance` field added
  5. `tests/test.sh` — already scaffolded, sources find_and_prove_verifier.sh
  6. `tests/find_and_prove_verifier.sh` — already scaffolded (copy of _shared/)

### Instruction.md Leakage Rules (CRITICAL)
- MUST describe only: error messages, user-reported behavior, test failures, observable symptoms
- MUST NOT contain: file paths, function/class names, line numbers, stack traces that pinpoint files, fix strategies
- Template:
  ```
  # Bug Investigation: [Symptom]
  **Repository:** [org/repo]
  **Task Type:** Find and Prove (write a regression test)
  ## Reported Issue
  [symptom only]
  ## Your Task
  1. Investigate the codebase to find the root cause
  2. Write a regression test at /workspace/regression_test.{ext}
  3. Your test must be self-contained and runnable with [test command]
  ## Constraints
  - Do NOT fix the bug — only write a test that demonstrates it
  - Your test must fail for the RIGHT reason
  - Test timeout: 60 seconds
  ```

### Dockerfile Pattern for Navprove
- Reuse the SWE-bench Pro base image (has the repo pre-cloned at the right commit)
- Add test framework: `RUN pip install pytest pytest-timeout` (Python) or ensure go test works (Go)
- Set WORKDIR to /workspace (or wherever the repo lives in the base image)
- The base image already has the repo at the buggy commit — the key is figuring out where

### Go Task Dockerfile Pattern
- Go base images already have Go toolchain — no need to install test frameworks
- Simpler than Python: just need FROM, mkdir /logs, /workspace symlink, WORKDIR, ENTRYPOINT
- No `pip install pytest` equivalent needed — `go test` is built into the Go toolchain
