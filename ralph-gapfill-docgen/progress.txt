## Codebase Patterns
- Doc gen tasks: benchmarks/ccb_docgen/{task_id}/ — new suite
- Three task subtypes: architecture_doc, api_reference, migration_guide
- Output format: /workspace/documentation.md (agent writes the doc)
- Scoring: weighted checklist via ground_truth.json — required_topics (each concept is a weighted fraction)
- Architecture docs: component responsibilities, data flow, extension points, error handling
- API references: method signatures, parameter semantics, error cases, usage examples from tests/internal callers
- Migration guides: breaking changes identified, migration steps per change, common pitfalls
- Migration tasks: Dockerfile provides two worktrees or directories (old version + new version)
- ground_truth.json: architecture uses required_topics[], file_references[], data_flow[], extension_points[]
- ground_truth.json: all per-category weights must sum to 1.0; top-level weights must sum to 1.0
- instruction.md: specify scope (which subsystem/API surface) but NOT file locations
- Config: configs/docgen_2config.sh (created by infra Ralph)
- MCP advantage: nls_search for component discovery, find_references for tracing interactions, compare_revisions for migration tasks
