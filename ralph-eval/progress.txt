## Codebase Patterns

### Module Layout
- All metrics/analysis code lives in `scripts/ccb_metrics/` as a Python package (no setup.py, uses sys.path)
- Scripts import via: `sys.path.insert(0, str(Path(__file__).resolve().parent))` then `from ccb_metrics.X import Y`
- New modules must be stdlib-only (no pip deps) except `anthropic` SDK for API backend
- Type hints and docstrings on all public functions; follow existing style in `ir_metrics.py`

### Ground Truth Infrastructure
- `configs/ground_truth_files.json` — cached registry mapping task_id -> {files[], source, confidence, benchmark}
- `scripts/ccb_metrics/ground_truth.py` — builds the registry; has per-benchmark extraction strategies
- Confidence levels: "high" (structured GT), "medium" (instruction extraction), "low" (regex fallback)
- 8 SDLC suites: ccb_build, ccb_debug, ccb_design, ccb_document, ccb_fix, ccb_secure, ccb_test, ccb_understand

### Benchmark Task Layout
- Tasks at: `benchmarks/ccb_{suite}/{task_name}/` with task.toml, instruction.md, environment/Dockerfile, tests/test.sh
- Ground truth files: tests/ground_truth.json, tests/expected_defects.json, tests/expected_changes.json
- Solution files: solution/solve.sh (patches)
- Verifier writes reward to: /logs/verifier/reward.txt (inside Docker container)
- Harbor uploads tests/ to /tests/ in container — use /tests/test.sh not /workspace/tests/test.sh

### Verifier Types (7 patterns)
1. IR_Checklist (ccb_build, ccb_design): verifier_lib.sh + keyword scoring
2. Diff_Similarity (ccb_fix/pytorch): verify_diff.py, line-level matching
3. Checklist/Pattern (ccb_test, ccb_secure, ccb_understand): regex keyword patterns
4. Find_and_Prove (ccb_debug): 2-phase majority voting
5. Test_Ratio (ccb_fix/swebenchpro): pytest pass/total
6. External (TAC, SWE-Perf): black-box Docker evaluator
7. Similarity (ccb_document): weighted keyword NL matching

### Run Directory Structure
- Official runs: runs/official/{suite}_{config}_{timestamp}/{config}/{batch_ts}/{task__hash}/result.json
- Staging runs: runs/staging/ (same structure)
- SKIP_PATTERNS: ["__broken_verifier", "validation_test", "archive", "__archived", "preamble_test_", "__v1_hinted"]
- Batch dirs match regex: ^\d{4}-\d{2}-\d{2}__\d{2}-\d{2}-\d{2}

### Existing Judge Schema
- schemas/judge_result.schema.json defines the output format
- scripts/ingest_judge_results.py reads judge_results/{benchmark}/{config}/ files
- Required fields: task_id, benchmark, config, judge_score (0.0-1.0), judge_model, judged_at
- Optional: rubric (dict of dimension:score), reasoning, confidence

### Discovery Module
- scripts/ccb_metrics/discovery.py — discover_runs(), _is_batch_dir(), _is_task_dir(), resolve_task_transcript_path()
- scripts/ccb_metrics/extractors.py — extract_tool_usage_from_transcript(), extract_tool_usage_from_trajectory()
- Transcript path: {task_dir}/agent/claude-code.txt (JSONL format)

### Key Conventions
- Config names: "baseline", "sourcegraph_full" (only two active configs)
- DIR_PREFIX_TO_SUITE mapping in ir_analysis.py and generate_manifest.py
- generate_manifest.py timestamp-based dedup: uses started_at from result.json
