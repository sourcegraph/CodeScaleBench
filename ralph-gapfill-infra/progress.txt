## Codebase Patterns
- Benchmark suites live in `benchmarks/ccb_{suite_name}/` with per-task subdirectories
- Each task has: task.toml, instruction.md, environment/Dockerfile, tests/test.sh, tests/ground_truth.json
- Config scripts in `configs/{suite}_2config.sh` follow investigation_2config.sh pattern
- `configs/_common.sh` provides: enforce_subscription_mode, setup_dual_accounts, run_paired_configs, ensure_fresh_token_all
- `scripts/generate_manifest.py` and `scripts/aggregate_status.py` both have DIR_PREFIX_TO_SUITE dicts that map run directory prefixes to suite names
- Investigation test.sh pattern: bash wrapper → embedded Python that reads ground_truth.json, scores report via regex pattern matching
- Suite CLAUDE.md files provide MCP search guidance specific to that task type
- selected_benchmark_tasks.json entries: {benchmark, task_id, repo, language, difficulty, mcp_benefit_score, ...}
- Archive location: benchmarks/archive/ for deprecated suites, configs/archive/ for deprecated configs
- Shared scorer template at `benchmarks/templates/weighted_checklist_scorer.sh` is parameterizable via REPORT_PATH env var
- New suites output paths: nlqa→`/logs/agent/investigation.md`, security→`/logs/agent/triage.md`, onboarding→`/logs/agent/onboarding.md`, docgen→`/workspace/documentation.md`
